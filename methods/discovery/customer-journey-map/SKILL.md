# Customer Journey Map (客户旅程地图)

## 目的

可视化用户从初次认知到成为忠实用户的完整旅程,识别关键时刻、痛点机会和情感曲线,帮助团队理解用户体验全貌,指导体验优化和产品改进。

## 核心概念

```
客户旅程地图结构
  |
  +-- 旅程阶段
  |    +-- 认知 -> 评估 -> 获取 -> 使用 -> 支持 -> 离开
  |
  +-- 用户行为
  |    +-- 每个阶段的具体行为
  |
  +-- 用户触点
  |    +-- 产品、渠道、人员等接触点
  |
  +-- 用户情感
  |    +-- 每个阶段的情感曲线
  |
  +-- 痛点机会
  |    +-- 体验痛点和改进机会
  |
  +-- 内部支持
       +-- 内部流程和资源支持
```

## 何时使用

- 产品发现阶段,理解用户完整体验
- 设计阶段,识别体验优化点
- 运营阶段,优化用户转化和留存
- 用户反馈集中,需要系统化分析
- 新产品上线,规划用户旅程

## 使用步骤

### Step 1: 准备阶段 - 明确旅程范围

**输入材料**:
1. 用户画像(已完成)
2. 用户访谈数据
3. 用户行为数据
4. 用户反馈数据

**准备工作**:
```
- 确定旅程起点和终点
- 确定目标用户画像
- 汇总用户研究数据
- 识别关键触点
```

**旅程范围定义**:
```
常见旅程范围:
- 完整旅程:从认知到离开的全过程
- 获取旅程:从认知到首次使用
- 使用旅程:从首次使用到深度使用
- 问题旅程:遇到问题到问题解决

示例:
本次旅程范围:完整旅程
起点:用户第一次听说产品
终点:用户成为忠实用户或离开
目标用户画像:李明(项目经理)
```

**输出格式**:
```
旅程范围定义

- 旅程类型:[完整/获取/使用/问题]
- 旅程起点:[起点描述]
- 旅程终点:[终点描述]
- 目标用户:[用户画像名称]
- 旅程时长:[预计时长范围]
```

### Step 2: 定义旅程阶段 (30-45 分钟)

**常见旅程阶段**:

#### 2.1 完整旅程阶段
```
阶段 1:认知(Awareness)
- 用户第一次听说产品
- 通过什么渠道了解到
- 初印象是什么

阶段 2:评估(Consideration)
- 用户开始考虑是否使用
- 对比哪些选择
- 决策依据是什么

阶段 3:获取(Acquisition)
- 用户决定使用
- 注册或购买过程
- 首次接触产品

阶段 4:使用(Usage)
- 用户开始使用产品
- 首次使用体验
- 日常使用模式

阶段 5:支持(Support)
- 用户遇到问题
- 寻求帮助过程
- 问题解决体验

阶段 6:离开(或留存)(Retention/Churn)
- 用户继续使用或离开
- 留存原因或离开原因
- 成为忠实用户或流失
```

**阶段细化方法**:
```
根据产品和用户特点细化阶段

示例:
项目管理工具旅程阶段:
1. 认知:听说项目管理工具
2. 评估:对比不同工具
3. 获取:注册试用
4. 学习:学习使用方法
5. 使用:日常项目管理
6. 支持:遇到问题求助
7. 扩展:推荐给团队
8. 留存:持续使用或离开
```

**输出格式**:
```
旅程阶段定义

| 阶段编号 | 阶段名称 | 阶段描述 | 预计时长 |
|----------|---------|---------|---------|
| 1 | 认知 | 第一次听说产品 | 1-7 天 |
| 2 | 评估 | 对比不同选择 | 1-2 周 |
| 3 | 获取 | 注册试用 | 1 天 |
| 4 | 学习 | 学习使用方法 | 1-2 周 |
| 5 | 使用 | 日常使用 | 持续 |
| 6 | 支持 | 遇到问题求助 | 按需 |
| 7 | 扩展 | 推荐给团队 | 1-3 月 |
| 8 | 留存 | 持续使用或离开 | 持续 |
```

### Step 3: 描述用户行为 (45-60 分钟)

**行为描述方法**:

#### 3.1 按阶段描述行为
```
详细描述用户在每个阶段的具体行为

示例:
阶段 1:认知
- 用户场景:项目延期,上级不满
- 用户行为:搜索"项目管理工具"
- 信息渠道:搜索引擎、同事推荐
- 初步印象:看到多个工具,不确定哪个合适

阶段 2:评估
- 用户场景:对比几个候选工具
- 用户行为:试用 3-4 个工具,查看评价
- 评估因素:易用性、功能、价格
- 决策困惑:不知道哪个最适合自己

阶段 3:获取
- 用户场景:决定试用我们的产品
- 用户行为:注册账号,下载 App
- 获取过程:填写邮箱、设置密码
- 首次接触:打开产品,看到欢迎页面

阶段 4:学习
- 用户场景:开始学习如何使用
- 用户行为:查看帮助文档,观看教程视频
- 学习过程:尝试创建第一个项目
- 学习感受:界面友好,但功能较多

阶段 5:使用
- 用户场景:开始日常使用
- 用户行为:每天查看任务、分配工作、更新进度
- 使用模式:早会前查看,下班前更新
- 使用感受:提高了效率,但偶尔遇到问题

阶段 6:支持
- 用户场景:遇到使用问题
- 用户行为:查看帮助文档,联系客服
- 问题类型:功能问题、使用疑问
- 支持体验:客服响应及时,问题解决

阶段 7:扩展
- 用户场景:觉得好用,推荐给团队
- 用户行为:向团队成员推荐,邀请注册
- 扩展过程:团队试用,逐渐接受
- 扩展感受:团队协作更顺畅

阶段 8:留存
- 用户场景:持续使用或考虑离开
- 用户行为:每天使用,或因问题考虑放弃
- 留存原因:提高效率,团队协作顺畅
- 离开原因:价格太高,功能不足
```

**输出格式**:
```
用户行为描述

### 阶段 1:认知
- 用户场景:[场景描述]
- 用户行为:[行为列表]
- 信息渠道:[渠道列表]
- 初步印象:[印象描述]

### 阶段 2:评估
...

### 阶段 3:获取
...

...
```

### Step 4: 识别用户触点 (45-60 分钟)

**触点分类**:

#### 4.1 产品触点
```
用户与产品直接接触的触点

示例:
- 网站/App:官网、下载页面
- 产品界面:首页、功能页面
- 功能触点:核心功能、辅助功能
- 内容触点:帮助文档、教程视频
```

#### 4.2 渠道触点
```
用户通过渠道接触产品的触点

示例:
- 搜索引擎:搜索结果、广告
- 社交媒体:微博、知乎、小红书
- 内容平台:博客、视频平台
- 应用商店:App Store、Google Play
```

#### 4.3 人员触点
```
用户与人员接触的触点

示例:
- 销售人员:销售咨询、演示
- 客服人员:客服支持、问题解决
- 团队成员:同事推荐、使用指导
```

**触点详细描述**:
```
按阶段描述触点

示例:
阶段 1:认知
触点:
- 搜索引擎:搜索"项目管理工具"
- 同事推荐:同事分享使用经验
- 广告投放:看到产品广告

阶段 2:评估
触点:
- 官网:查看产品介绍
- 试用页面:注册试用
- 用户评价:查看 App Store 评论
- 竞品对比:对比其他工具

阶段 3:获取
触点:
- 注册页面:填写注册信息
- 下载页面:下载 App 或打开网页
- 欢迎页面:首次打开看到的页面

阶段 4:学习
触点:
- 新手引导:产品内引导流程
- 帮助文档:查看使用说明
- 教程视频:观看功能演示
- 社区论坛:查看用户分享

阶段 5:使用
触点:
- 产品界面:日常使用的页面
- 功能入口:各功能模块
- 通知提醒:任务提醒、进度更新
- 数据展示:项目数据可视化

阶段 6:支持
触点:
- 帮助中心:查找解决方案
- 客服咨询:在线客服、电话客服
- 用户社区:提问和解答
- 问题反馈:提交问题工单

阶段 7:扩展
触点:
- 分享功能:分享给团队成员
- 邀请入口:邀请注册功能
- 团队管理:添加团队成员

阶段 8:留存
触点:
- 续费提醒:续费通知
- 功能更新:新功能介绍
- 用户活动:用户社区活动
```

**输出格式**:
```
用户触点描述

| 阶段 | 触点类型 | 触点名称 | 触点描述 |
|------|---------|---------|---------|
| 1 | 渠道 | 搜索引擎 | 搜索"项目管理工具" |
| 1 | 人员 | 同事推荐 | 同事分享使用经验 |
| 2 | 产品 | 官网 | 查看产品介绍 |
| 2 | 产品 | 试用页面 | 注册试用 |
| 2 | 渠道 | 用户评价 | 查看 App Store 评论 |
...
```

### Step 5: 绘制情感曲线 (30-45 分钟)

**情感曲线定义**:
```
用户在每个阶段的情感状态曲线

情感状态:
- 高:满意、开心、兴奋
- 中:平静、一般、无感
- 低:困惑、沮丧、愤怒

绘制方法:
- 基于用户访谈和反馈
- 标注情感转折点
- 识别情感驱动因素
```

**情感曲线绘制**:
```
示例:
阶段 1:认知
情感状态:中(平静,开始寻找解决方案)
情感驱动:项目延期压力,开始主动寻找

阶段 2:评估
情感状态:低 -> 中(困惑 -> 稍有方向)
情感驱动:不知道选哪个 -> 试用后有方向

阶段 3:获取
情感状态:中 -> 高(期待 -> 期待)
情感驱动:注册成功,期待使用

阶段 4:学习
情感状态:中(好奇但有点困惑)
情感驱动:功能多,需要学习

阶段 5:使用
情感状态:中 -> 高(逐渐习惯 -> 满意)
情感驱动:使用顺畅,效率提升

阶段 6:支持
情感状态:低 -> 高(遇到问题 -> 问题解决)
情感驱动:遇到问题沮丧 -> 客服帮助满意

阶段 7:扩展
情感状态:高(满意,愿意分享)
情感驱动:觉得好用,推荐给团队

阶段 8:留存
情感状态:高(持续满意)
情感驱动:持续高效,团队协作顺畅

情感曲线图:
[绘制情感状态曲线图,标注高点和低点]
```

**输出格式**:
```
情感曲线描述

| 阶段 | 情感状态 | 情感分数(1-5) | 情感驱动因素 |
|------|---------|--------------|-------------|
| 1 | 中 | 3 | 开始寻找解决方案 |
| 2 | 低 -> 中 | 2 -> 3 | 困惑 -> 有方向 |
| 3 | 中 -> 高 | 3 -> 4 | 注册成功,期待使用 |
| 4 | 中 | 3 | 功能多,需要学习 |
| 5 | 中 -> 高 | 3 -> 4 | 使用顺畅,效率提升 |
| 6 | 低 -> 高 | 2 -> 4 | 问题 -> 解决 |
| 7 | 高 | 4 | 满意,愿意分享 |
| 8 | 高 | 4 | 持续满意 |
```

### Step 6: 识别痛点机会 (45-60 分钟)

**痛点识别方法**:

#### 6.1 按阶段识别痛点
```
识别每个阶段的体验痛点

示例:
阶段 1:认知
痛点:
- 不知道有哪些工具可选
- 搜索结果太多,难以判断
- 信息不够清晰直观

阶段 2:评估
痛点:
- 试用功能受限,无法全面了解
- 对比信息不够透明
- 不知道哪个最适合自己的场景

阶段 3:获取
痛点:
- 注册流程太繁琐
- 需要填写太多信息
- 下载速度慢或失败

阶段 4:学习
痛点:
- 功能介绍不够直观
- 学习成本高,时间不够
- 没有人指导,自己摸索困难

阶段 5:使用
痛点:
- 部分功能不够流畅
- 响应速度偶尔较慢
- 数据同步有时出错

阶段 6:支持
痛点:
- 帮助文档不够详细
- 客服响应时间较长
- 问题解决不够彻底

阶段 7:扩展
痛点:
- 邀请流程不够简单
- 团队成员接受度不一
- 需要说服团队成员

阶段 8:留存
痛点:
- 价格上涨,预算不够
- 功能更新不符合需求
- 竞品有更好的功能
```

**机会识别方法**:

#### 6.2 按痛点识别机会
```
针对痛点提出改进机会

示例:
痛点:不知道有哪些工具可选
机会:提供工具对比页面,清晰展示差异

痛点:试用功能受限
机会:开放更多试用功能,延长试用期

痛点:注册流程繁琐
机会:简化注册流程,减少必填信息

痛点:学习成本高
机会:增加新手引导,提供视频教程

痛点:帮助文档不够详细
机会:完善帮助文档,增加视频指导

痛点:邀请流程复杂
机会:简化邀请流程,一键分享链接
```

**输出格式**:
```
痛点机会识别

| 阶段 | 痛点描述 | 痛点程度 | 改进机会 | 优先级 |
|------|---------|---------|---------|--------|
| 1 | 不知道有哪些工具可选 | 高 | 提供工具对比页面 | P1 |
| 2 | 试用功能受限 | 高 | 开放更多试用功能 | P0 |
| 3 | 注册流程繁琐 | 中 | 简化注册流程 | P1 |
| 4 | 学习成本高 | 高 | 增加新手引导 | P0 |
| 5 | 部分功能不流畅 | 中 | 优化功能体验 | P2 |
| 6 | 帮助文档不够详细 | 中 | 完善帮助文档 | P1 |
| 7 | 邀请流程复杂 | 低 | 简化邀请流程 | P2 |
| 8 | 价格上涨 | 中 | 提供灵活定价 | P2 |
```

### Step 7: 绘制旅程地图 (45-60 分钟)

**地图绘制方法**:

#### 7.1 横向布局
```
横向布局旅程阶段

结构:
阶段 1 -> 阶段 2 -> 阶段 3 -> ...

每列包含:
- 用户行为
- 用户触点
- 用户情感
- 痛点机会
```

#### 7.2 情感曲线可视化
```
在地图上方绘制情感曲线

方法:
- 用折线图展示情感变化
- 标注情感高点和低点
- 用颜色区分情感状态

颜色建议:
- 绿色:高情感(满意)
- 黄色:中情感(平静)
- 红色:低情感(痛点)
```

#### 7.3 内部支持映射
```
在地图下方标注内部支持

内容:
- 内部流程支持
- 内部资源支持
- 内部团队支持

示例:
阶段 3:获取
内部支持:
- 注册流程优化
- 用户引导团队
- 客服支持团队

阶段 4:学习
内部支持:
- 帮助文档维护
- 教程视频制作
- 用户社区运营
```

**输出格式**:
```
客户旅程地图可视化

阶段 1 | 阶段 2 | 阶段 3 | 阶段 4 | 阶段 5 | 阶段 6 | 阶段 7 | 阶段 8

每列内容:
- 用户行为:[行为描述]
- 用户触点:[触点列表]
- 用户情感:[情感状态]
- 痛点机会:[痛点 + 机会]

上方:情感曲线图

下方:内部支持映射
```

## 常见错误

- 基于假设而非真实数据绘制旅程
- 旅程阶段过于笼统或过于细节
- 忽视情感曲线和痛点识别
- 只关注用户行为,忽视内部支持
- 旅程地图过于复杂,难以使用
- 没有基于旅程地图制定改进计划

## 与其他方法的关系

- 前置方法: 用户画像、用户访谈
- 并行方法: 用户触点分析
- 后置方法: 体验优化、流程改进
