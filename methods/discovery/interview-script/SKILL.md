# Interview Script (用户访谈脚本)

## 目的

创建结构化的用户访谈脚本，通过深度访谈挖掘用户真实需求、痛点和行为模式，避免诱导性提问和推销，获取可靠的用户洞察。

## 核心概念

```
用户访谈脚本结构
  │
  ├─ 暖场环节（5 分钟）
  │   └─ 建立信任，了解背景
  │
  ├─ 核心探索环节（20-30 分钟）
  │   ├─ 现状探索
  │   ├─ 痛点挖掘
  │   ├─ 过去行为
  │   └─ 决策过程
  │
  ├─ JTBD 探究环节（10-15 分钟）
  │   ├─ 四个维度：功能、情感、社会、现状
  │   └─ 进展历程：初念 → 被动研究 → 主动研究 → 决策
  │
  └─ 收尾环节（5 分钟）
      └─ 总结确认，后续联系
```

## 何时使用

- ✅ 产品发现阶段，需要理解用户需求
- ✅ 识别用户痛点和机会
- ✅ 验证产品假设
- ✅ 了解用户决策过程
- ✅ 探索用户行为模式

## 使用步骤

### Step 1: 准备阶段 - 明确访谈目标

**输入材料**:
1. **研究问题**: 我们想了解什么？
2. **目标用户**: 我们要访谈谁？
3. **产品假设**: 我们有哪些假设需要验证？

**准备问题**:
```
- 我们想通过访谈了解什么？
- 我们对用户有什么假设？
- 哪些信息对产品决策最重要？
- 如何避免诱导用户回答？
```

**The Mom Test 三原则**:
```
1. 谈论过去的行为，而非未来的计划
   ❌ 「你会使用这个功能吗？」
   ✅ 「上次遇到这个问题时，你是怎么解决的？」

2. 关注具体事实，而非泛泛观点
   ❌ 「你觉得这个功能怎么样？」
   ✅ 「上次使用类似功能时，具体发生了什么？」

3. 不要推销你的想法
   ❌ 「如果我们做了这个功能，你会喜欢吗？」
   ✅ 「你现在是如何解决这个问题的？」
```

**输出格式**:
```
## 访谈目标明确

- 研究问题：[想了解什么]
- 目标用户：[用户画像]
- 关键假设：[需要验证的假设]
- 访谈时长：[总时长]
- 访谈方式：[面对面/视频/电话]
```

### Step 2: 设计暖场环节 (5 分钟)

**目的**: 建立信任，了解用户背景

**问题设计**:

#### 2.1 自我介绍
```
话术模板：

「你好，我是 [姓名]，在 [公司] 工作。非常感谢你愿意花时间参与这次访谈。

这次访谈主要是想了解你在 [领域/场景] 的一些经历和想法。我们的产品还在探索阶段，你的真实反馈对我们非常重要。

访谈大约需要 [时长]，全程没有对错之分，我们只是想了解你的真实想法。访谈内容我们会严格保密，仅用于产品改进。

如果你同意，我们会录音以便后续整理，你觉得可以吗？」

要点：
- 明确身份和目的
- 强调没有对错之分
- 确保隐私保护
- 征得录音同意
```

#### 2.2 背景了解
```
问题模板：

1. 「可以先简单介绍一下你自己吗？比如你的工作、日常主要做什么？」

2. 「你使用 [产品类型/服务] 多长时间了？」

3. 「平时主要在什么场景下使用这类产品？」

注意：
- 开放式问题，让用户自由发挥
- 不限制回答方向
- 观察用户提到的关键词
```

**输出格式**:
```
## 暖场环节脚本

[开场白]
...

[背景问题]
1. ...
2. ...
3. ...

预计时长：5 分钟
```

### Step 3: 设计核心探索环节 (20-30 分钟)

**目的**: 深入了解用户现状、痛点、行为

**问题设计框架**:

#### 3.1 现状探索
```
目的：了解用户当前如何解决问题

问题模板：

1. 「你能描述一下最近一次遇到 [问题] 是什么时候吗？」

2. 「当时具体发生了什么？能详细说说吗？」

3. 「这个问题对你造成了什么影响？」

4. 「这个问题出现的频率高吗？」

The Mom Test 技巧：
- 追问具体案例
- 关注时间和场景
- 了解问题影响
```

#### 3.2 痛点挖掘
```
目的：识别用户最痛的点

问题模板：

1. 「在解决这个问题的过程中，最让你困扰的是什么？」

2. 「有没有哪个环节让你特别不爽？」

3. 「如果能改变一件事，你最希望改变什么？」

4. 「这个问题不解决会怎么样？」

深度挖掘技巧：
- 使用「为什么」追问
- 询问具体感受
- 了解问题优先级
```

#### 3.3 过去行为
```
目的：了解用户真实的行为模式

问题模板：

1. 「你是怎么发现这个问题的？」

2. 「发现后你做了什么？」

3. 「后来呢？」

4. 「最后是怎么解决的？」

5. 「解决后效果如何？」

The Mom Test 核心：
- 关注过去的具体行为
- 追问完整的故事
- 了解真实决策过程
```

#### 3.4 决策过程
```
目的：了解用户的决策逻辑

问题模板：

1. 「当时考虑过哪些解决方案？」

2. 「为什么最终选择了这个方案？」

3. 「有没有考虑过其他方案？为什么放弃了？」

4. 「是什么让你下定决心做这个决策的？」

5. 「如果重来一次，你还会做同样的选择吗？」

决策洞察：
- 了解决策触发点
- 识别关键决策因素
- 发现决策障碍
```

**输出格式**:
```
## 核心探索环节脚本

### 现状探索（5-8 分钟）
1. ...
2. ...

### 痛点挖掘（5-8 分钟）
1. ...
2. ...

### 过去行为（5-8 分钟）
1. ...
2. ...

### 决策过程（5-8 分钟）
1. ...
2. ...

预计时长：20-30 分钟
```

### Step 4: 设计 JTBD 探究环节 (10-15 分钟)

**目的**: 从 Jobs-to-be-Done 视角深入理解用户需求

**JTBD 四个维度**:

#### 4.1 功能维度（Functional Job）
```
问题：用户想完成什么具体任务？

问题模板：

1. 「你使用 [产品/服务] 主要想达成什么目标？」

2. 「具体想解决什么问题？」

3. 「完成后应该是什么样子？」

示例：
用户：我想用项目管理工具
功能 Job：协调团队任务，按时完成项目
```

#### 4.2 情感维度（Emotional Job）
```
问题：用户想获得什么情感满足？

问题模板：

1. 「使用 [产品/服务] 时，你希望有什么感觉？」

2. 「避免什么感觉？」

3. 「完成后你会有什么感受？」

示例：
用户：我想用项目管理工具
情感 Job：减少焦虑，感到掌控感
```

#### 4.3 社会维度（Social Job）
```
问题：用户希望在他人眼中是什么形象？

问题模板：

1. 「使用 [产品/服务] 时，你希望别人怎么看你？」

2. 「不希望别人怎么看你？」

3. 「这对你重要吗？为什么？」

示例：
用户：我想用项目管理工具
社会 Job：看起来专业、有条理
```

#### 4.4 现状维度（Current State）
```
问题：用户现在处于什么状态？

问题模板：

1. 「现在是怎么做的？」

2. 「现在的解决方案有什么问题？」

3. 「为什么现在的方法不够好？」

示例：
用户：我想用项目管理工具
现状：用 Excel 管理，混乱、易出错
```

**JTBD 进展历程（Progress Timeline）**:

```
问题：用户从产生想法到做出决策的完整过程

问题模板：

1. 初念（First Thought）：
   「是什么让你第一次想到要 [做某事]？」

2. 被动研究（Passive Research）：
   「之后你做了什么？有去了解相关信息吗？」

3. 主动研究（Active Research）：
   「你是怎么寻找解决方案的？考虑了哪些选项？」

4. 决策（Decision）：
   「最终是什么让你做出选择的？」

5. 使用（Usage）：
   「使用后感觉如何？有什么惊喜或失望吗？」

示例：
用户购买项目管理工具的历程：
- 初念：项目延期，团队协作混乱
- 被动研究：同事推荐，看到广告
- 主动研究：试用 3-4 个工具，对比功能
- 决策：选了 JIRA，因为功能最全
- 使用：学习成本高，但功能强大
```

**输出格式**:
```
## JTBD 探究环节脚本

### 功能维度（3-4 分钟）
1. ...
2. ...

### 情感维度（3-4 分钟）
1. ...
2. ...

### 社会维度（2-3 分钟）
1. ...
2. ...

### 现状维度（2-3 分钟）
1. ...
2. ...

### 进展历程（5-8 分钟）
1. 初念：...
2. 被动研究：...
3. 主动研究：...
4. 决策：...
5. 使用：...

预计时长：10-15 分钟
```

### Step 5: 设计收尾环节 (5 分钟)

**目的**: 总结确认，建立后续联系

**问题设计**:

#### 5.1 总结确认
```
话术模板：

「非常感谢你的分享，我总结一下你刚才提到的关键点...」

「[重复用户提到的关键痛点、需求、行为]」

「我理解得对吗？有没有遗漏或需要补充的？」

要点：
- 总结关键洞察
- 确认理解准确
- 补充遗漏信息
```

#### 5.2 开放提问
```
话术模板：

「还有什么想分享的，或者觉得重要但我没有问到的吗？」

「最后有没有什么问题想问我的？」

要点：
- 给用户表达机会
- 可能获得意外洞察
- 保持开放态度
```

#### 5.3 后续联系
```
话术模板：

「非常感谢你的时间，你的反馈对我们非常有价值。」

「如果后续有产品更新或需要再次访谈，我们可以再联系你吗？」

「这是我的联系方式，如果有任何问题随时联系我。」

要点：
- 表达感谢
- 建立长期关系
- 提供联系方式
```

**输出格式**:
```
## 收尾环节脚本

[总结确认]
...

[开放提问]
...

[后续联系]
...

预计时长：5 分钟
```

```markdown
## 访谈注意事项

### DO（应该做）
- ✅ 关注过去的具体行为
- ✅ 追问具体案例和细节
- ✅ 问「为什么」和「怎么」
- ✅ 保持沉默，让用户多说
- ✅ 观察用户的情绪和表情

### DON'T（不应该做）
- ❌ 问「你会不会使用这个功能？」
- ❌ 问「你觉得这个功能怎么样？」
- ❌ 推销你的想法或产品
- ❌ 引导用户回答你想听到的
- ❌ 打断用户的话

## 访谈准备清单
- [ ] 确认访谈时间和地点
- [ ] 准备录音设备
- [ ] 打印访谈脚本
- [ ] 准备用户激励（如有）
- [ ] 确认访谈环境安静
- [ ] 测试网络（如视频访谈）
```

## 常见错误

- ❌ 问未来计划而非过去行为
- ❌ 问观点而非具体事实
- ❌ 推销产品或想法
- ❌ 引导性提问
- ❌ 问太多封闭式问题
- ❌ 访谈时间过长
- ❌ 不追问细节
- ❌ 打断用户

## 与其他方法的关系

- **前置方法**: 用户招募、用户画像
- **并行方法**: 无
- **后置方法**: summarize-interview（访谈总结）
