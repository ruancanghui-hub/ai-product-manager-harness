# Brainstorm Experiments - New Product (新产品实验设计)

## 目的

针对新产品或新业务线的关键假设，设计精益创业实验（Pretotype），在不投入大量开发资源的情况下，快速验证市场需求和用户支付意愿，避免在错误方向上浪费资源。

## 核心概念

```
Pretotype 实验类型
  │
  ├─ 落地页测试（Landing Page）
  │   └─ 验证用户是否对产品感兴趣
  │
  ├─ 解说视频（Explainer Video）
  │   └─ 用视频展示产品概念，测试用户反应
  │
  ├─ 预售测试（Pre-order）
  │   └─ 用户预付定金，验证支付意愿
  │
  ├─ 众筹测试（Crowdfunding）
  │   └─ 在众筹平台验证市场接受度
  │
  ├─ Wizard of Oz（绿野仙踪）
  │   └─ 人工模拟自动化服务，验证需求
  │
  ├─ Concierge MVP（礼宾 MVP）
  │   └─ 人工提供完整服务，验证流程
  │
  └─ 烟雾测试（Smoke Test）
      └─ 应用商店截图、广告投放测试
```

## 何时使用

- ✅ 新产品从 0 到 1 的市场验证
- ✅ 不确定是否有市场需求
- ✅ 不确定用户是否愿意付费
- ✅ 寻求投资前，需要验证关键假设
- ✅ 资源有限，需要低成本快速验证

## 使用步骤

### Step 1: 准备阶段 - 明确验证目标

**输入材料**:
1. **关键假设**: 从 prioritize-assumptions 得到的关键路径假设
2. **产品概念**: 产品是什么，解决什么问题
3. **价值主张**: 为什么用户会选择我们
4. **验证预算**: 可用的时间、金钱、人力

**准备问题**:
```
- 我们要验证的最关键假设是什么？
- 如果这个假设不成立，产品会失败吗？
- 我们能用最小成本验证吗？
- 如何判断验证成功或失败？
```

**输出格式**:
```
## 验证目标明确

- 产品名称：[名称]
- 核心价值主张：[一句话描述]
- 关键假设：[假设描述]
- 验证预算：时间 X 周，预算 Y 元
- 成功标准：[明确的成功指标]
```

### Step 2: 选择 Pretotype 类型 (30-45 分钟)

**Pretotype 决策树**:

#### 2.1 需求验证类 Pretotype
```
假设：用户有这个问题/需求

Pretotype 类型：
├─ 用户访谈（深度理解问题）
├─ 落地页测试（量化兴趣度）
├─ 解说视频（展示概念测试反应）
└─ 数据分析（市场规模验证）

示例：
产品：宠物健康追踪 App
假设：宠物主人担心宠物健康，愿意追踪
Pretotype：
- 落地页测试：制作产品介绍页面，看注册转化率
- 解说视频：制作 2 分钟视频，看播放和互动
```

#### 2.2 解决方案验证类 Pretotype
```
假设：我们的解决方案是最佳方案

Pretotype 类型：
├─ Wizard of Oz（人工模拟自动化）
├─ Concierge MVP（人工提供完整服务）
├─ 原型测试（Figma 原型）
└─ 假门测试（测试功能兴趣）

示例：
产品：AI 客服机器人
假设：AI 能解决大部分客服问题
Pretotype：
- Wizard of Oz：人工扮演 AI 回答问题，验证需求
- 原型测试：Figma 原型测试用户交互
```

#### 2.3 支付意愿验证类 Pretotype
```
假设：用户愿意为此付费

Pretotype 类型：
├─ 预售测试（真实预付）
├─ 众筹测试（Kickstarter 等）
├─ 价格测试（不同价格点）
└─ 假门测试（付费功能入口）

示例：
产品：宠物健康追踪 App 高级版
假设：用户愿意为高级功能每月付 30 元
Pretotype：
- 预售测试：推出早鸟价，看预售订单
- 众筹测试：Kickstarter 上众筹
```

#### 2.4 市场规模验证类 Pretotype
```
假设：市场规模足够大

Pretotype 类型：
├─ 数据调研（行业报告）
├─ 广告投放测试（获客成本）
├─ SEO 测试（搜索量）
└─ 社交媒体测试（话题热度）

示例：
产品：宠物健康追踪 App
假设：中国宠物健康市场足够大
Pretotype：
- 数据调研：宠物医疗市场规模
- 广告投放：测试 CAC 和转化率
```

**输出格式**:
```
## Pretotype 类型选择

| 假设ID | 假设描述 | 推荐 Pretotype | 次选 Pretotype |
|--------|---------|--------------|---------------|
| VAL-01 | 用户有 X 问题 | 落地页测试 | 解说视频 |
| VAL-02 | 方案能解决 | Wizard of Oz | 原型测试 |
| VIA-01 | 用户愿意付费 | 预售测试 | 众筹测试 |
| MAR-01 | 市场足够大 | 广告投放测试 | 数据调研 |
```

### Step 3: 设计具体 Pretotype (60-90 分钟)

**Pretotype 设计框架**:

#### 3.1 落地页测试
```
Pretotype 设计模板：

Pretotype 名称：落地页测试 - [产品名称]
验证假设：用户对产品感兴趣，愿意留下联系方式

页面结构：
1. 头部：
   - Logo 和产品名称
   - 一句话价值主张

2. 英雄区域：
   - 核心价值主张（大标题）
   - 副标题（补充说明）
   - CTA 按钮：「立即注册」/「加入等待名单」
   - 产品截图或概念图

3. 功能介绍：
   - 3-5 个核心功能
   - 每个功能配图 + 简短描述

4. 使用场景：
   - 2-3 个典型使用场景
   - 用户故事形式

5. 社会证明：
   - 用户评价（如有）
   - 媒体报道（如有）

6. CTA 区域：
   - 再次强调价值主张
   - CTA 按钮
   - 隐私承诺

数据指标：
- 页面浏览量（PV/UV）
- 注册转化率
- 跳出率
- 停留时间

成功标准：
- 转化率 > 5%（行业基准）
- 跳出率 < 60%
- 停留时间 > 1 分钟

流量来源：
- 广告投放（Google/Facebook/抖音等）
- SEO（搜索引擎优化）
- 社交媒体（微博/小红书/知乎等）
- 内容营销（博客/视频等）

资源需求：
- 时间：1 周（页面制作） + 1-2 周（投放）
- 金钱：5000 元（广告投放）
- 人力：设计师 1 天 + 运营 2 天

执行计划：
D1-D3：页面设计和开发
D4-D5：测试和优化
W2-W3：广告投放
W4：数据分析
```

#### 3.2 解说视频测试
```
Pretotype 设计模板：

Pretotype 名称：解说视频 - [产品名称]
验证假设：用户理解产品价值后感兴趣

视频结构（2-3 分钟）：
1. 问题引入（15 秒）
   - 展示用户面临的痛点

2. 解决方案（30 秒）
   - 引出产品概念
   - 展示核心价值

3. 功能演示（60 秒）
   - 演示 2-3 个核心功能
   - 使用场景展示

4. 使用效果（30 秒）
   - 用户使用前后的对比
   - 成功案例（如有）

5. CTA（15 秒）
   - 邀请注册/了解更多

视频风格：
- 动画解说（成本低，效果好）
- 真人演示（真实感强）
- 产品录屏（适合软件产品）

分发渠道：
- YouTube/B 站
- 社交媒体
- 广告投放

数据指标：
- 播放量
- 完播率
- 点击率
- 转化率

成功标准：
- 完播率 > 40%
- 点击率 > 2%
- 转化率 > 3%

资源需求：
- 时间：1 周（视频制作） + 1-2 周（投放）
- 金钱：3000 元（视频制作 + 投放）
- 人力：设计师 2 天 + 运营 1 天

执行计划：
D1-D3：脚本和分镜
D4-D5：视频制作
W2-W3：分发和投放
W4：数据分析
```

#### 3.3 预售测试
```
Pretotype 设计模板：

Pretotype 名称：预售测试 - [产品名称]
验证假设：用户愿意为产品付费

预售页面结构：
1. 产品介绍：
   - 产品是什么
   - 解决什么问题
   - 核心功能

2. 预售优惠：
   - 早鸟价（低于正式价格）
   - 预售专属福利
   - 限量名额

3. 交付承诺：
   - 预计交付时间
   - 交付内容
   - 退款政策

4. 信任建立：
   - 团队背景
   - 项目进展
   - 用户评价（如有）

5. 购买按钮：
   - 清晰的 CTA
   - 多种支付方式

预售策略：
- 早鸟价：正式价格的 50-70%
- 名额限制：限量 100-500 名
- 时间限制：限时 2-4 周
- 退款承诺：如未交付全额退款

数据指标：
- 预售订单量
- 预售收入
- 转化率
- 用户反馈

成功标准：
- 预售订单量 > 100 单
- 预售收入 > 10000 元
- 转化率 > 2%

资源需求：
- 时间：1 周（页面制作） + 2-4 周（预售）
- 金钱：2000 元（页面制作 + 宣传）
- 人力：PM 2 天 + 运营 3 天

执行计划：
D1-D3：预售页面制作
D4-D5：支付系统对接
W2-W5：预售宣传
W6：数据分析和用户沟通
```

#### 3.4 众筹测试
```
Pretotype 设计模板：

Pretotype 名称：众筹测试 - [产品名称]
验证假设：市场接受度足够，能获得支持

众筹平台选择：
- 国际：Kickstarter, Indiegogo
- 国内：造点新货, 摩点, 京东众筹

众筹页面结构：
1. 项目标题：
   - 清晰的产品定位
   - 吸引人的口号

2. 项目视频：
   - 2-3 分钟解说视频
   - 展示产品和团队

3. 项目详情：
   - 产品介绍
   - 功能特点
   - 使用场景
   - 规格参数

4. 回报档位：
   - 3-5 个不同价位
   - 每个档位的回报内容
   - 早鸟优惠

5. 项目进展：
   - 当前进展
   - 时间规划
   - 风险说明

6. 团队介绍：
   - 团队成员
   - 过往经验

众筹目标：
- 筹款目标：[金额]
- 时间周期：30-45 天
- 达成标准：目标金额的 100%

数据指标：
- 支持者数量
- 筹款金额
- 平均支持金额
- 社交媒体传播

成功标准：
- 筹款目标达成
- 支持者 > 200 人
- 平均支持金额 > 100 元

资源需求：
- 时间：2 周（准备） + 30-45 天（众筹）
- 金钱：5000 元（视频制作 + 宣传）
- 人力：PM 3 天 + 运营 1 周

执行计划：
W1-W2：众筹页面制作
W3：众筹预热
W4-W10：众筹进行
W11：项目总结
```

#### 3.5 Wizard of Oz 测试
```
Pretotype 设计模板：

Pretotype 名称：Wizard of Oz - [产品名称]
验证假设：自动化服务能满足用户需求

实施方式：
- 用户界面：看起来像真实的自动化产品
- 后台操作：人工手动处理用户请求
- 响应时间：模拟真实产品的响应时间

示例：
产品：AI 简历优化工具
界面：用户上传简历，点击「AI 优化」
后台：人工 HR 专家优化简历
输出：优化后的简历

流程设计：
1. 用户访问界面
2. 用户提交请求
3. 后台人工处理
4. 返回处理结果
5. 收集用户反馈

数据指标：
- 用户使用量
- 任务完成率
- 用户满意度
- 重复使用率

成功标准：
- 用户满意度 > 4 分（5 分制）
- 重复使用率 > 30%
- 用户愿意推荐

资源需求：
- 时间：1 周（界面制作） + 2-4 周（测试）
- 金钱：1000 元（界面制作）
- 人力：开发 2 天 + 运营 1 周（人工处理）

执行计划：
D1-D3：界面制作
D4-D5：流程测试
W2-W5：用户测试
W6：数据分析
```

#### 3.6 Concierge MVP 测试
```
Pretotype 设计模板：

Pretotype 名称：Concierge MVP - [产品名称]
验证假设：完整服务流程能满足用户需求

实施方式：
- 不开发产品，用人工提供完整服务
- 体验与最终产品一致
- 收集用户反馈和行为数据

示例：
产品：个性化旅行规划 App
服务：人工旅行顾问提供定制行程
流程：
1. 用户填写需求问卷
2. 顾问电话沟通详细需求
3. 顾问制作定制行程
4. 用户获得行程 PDF
5. 收集用户反馈

流程设计：
1. 用户招募
2. 需求收集
3. 服务提供
4. 结果交付
5. 反馈收集

数据指标：
- 服务使用量
- 服务满意度
- 推荐意愿（NPS）
- 支付意愿

成功标准：
- 服务满意度 > 4.5 分（5 分制）
- NPS > 50
- 70% 用户愿意付费

资源需求：
- 时间：2-4 周（服务提供）
- 金钱：1000 元（工具和激励）
- 人力：PM 2 周（服务提供）

执行计划：
W1：服务流程设计
W2-W5：用户服务
W6：数据分析和总结
```

#### 3.7 烟雾测试
```
Pretotype 设计模板：

Pretotype 名称：烟雾测试 - [产品名称]
验证假设：用户对产品概念感兴趣

测试方式：
1. 应用商店截图测试：
   - 制作 App Store/Google Play 截图
   - 观察转化率和下载意愿

2. 广告投放测试：
   - 制作产品概念广告
   - 观察点击率和转化率

3. 社交媒体测试：
   - 发布产品概念内容
   - 观察互动和关注

应用商店截图测试：
- 制作 3-5 张 App 截图
- 包含功能介绍和价值主张
- 提交到应用商店（不上架）
- 观察页面转化率

数据指标：
- 点击率（CTR）
- 注册转化率
- 成本 per 安装（CPI）

成功标准：
- CTR > 1%
- 转化率 > 5%
- CPI < 10 元

资源需求：
- 时间：1 周（制作） + 1-2 周（测试）
- 金钱：3000 元（投放）
- 人力：设计师 1 天 + 运营 1 天

执行计划：
D1-D3：素材制作
W2-W3：投放测试
W4：数据分析
```

**输出格式**:
```
## Pretotype 设计详情

### Pretotype 名称：[名称]
- 验证假设：[假设描述]
- Pretotype 类型：[类型]
- 测试时长：[时间]
- 成功标准：[明确标准]
- 资源需求：[时间/金钱/人力]
- 执行计划：[详细计划]
```

### Step 4: 评估 Pretotype 可行性 (20-30 分钟)

**评估维度**:

#### 4.1 技术可行性
```
问题：我们能实现这个 Pretotype 吗？

评估：
- 需要哪些技术实现？
- 技术难度多大？
- 有现成工具支持吗？

示例：
Pretotype：落地页测试
评估：
- 技术需求：网站制作
- 技术难度：低
- 现成工具：WordPress, Webflow, Unbounce
- 可行性：高
```

#### 4.2 资源可行性
```
问题：有足够的资源吗？

评估：
- 时间是否充足？
- 预算是否充足？
- 人力是否充足？

示例：
Pretotype：预售测试
评估：
- 时间：2-4 周（充足）
- 预算：2000 元（充足）
- 人力：PM 2 天 + 运营 3 天（需要协调）
- 可行性：中
```

#### 4.3 验证有效性
```
问题：这个 Pretotype 能有效验证假设吗？

评估：
- 能否获得真实用户反馈？
- 样本是否代表性？
- 结果是否可靠？

示例：
Pretotype：落地页测试
评估：
- 用户行为真实：是
- 样本代表性：取决于流量来源
- 结果可靠性：转化率数据客观
- 验证有效性：高
```

**输出格式**:
```
## Pretotype 可行性评估

| Pretotype ID | 名称 | 技术可行性 | 资源可行性 | 验证有效性 | 综合评估 |
|-------------|------|-----------|-----------|-----------|---------|
| PRE-01 | 落地页测试 | 高 | 高 | 高 | ✅ 可执行 |
| PRE-02 | 预售测试 | 高 | 中 | 高 | ⚠️ 需调整 |
| PRE-03 | 众筹测试 | 中 | 中 | 高 | ⚠️ 需调整 |
```

### Step 5: 制定验证计划 (20-30 分钟)

**计划内容**:

#### 5.1 验证优先级
```
验证优先级 = 假设关键性 × Pretotype 可行性

示例：
Pretotype A：关键假设 × 高可行性 = 最高优先级
Pretotype B：重要假设 × 高可行性 = 高优先级
Pretotype C：关键假设 × 低可行性 = 寻找替代方案
```

## 常见错误

- ❌ Pretotype 过于复杂，成本过高
- ❌ 成功标准不明确，无法判断结果
- ❌ 样本量不足，结果不可靠
- ❌ 流量来源不精准，用户非目标群体
- ❌ 过度承诺，导致用户失望
- ❌ 忽视用户反馈，只看数据

## 与其他方法的关系

- **前置方法**: prioritize-assumptions（假设排序）
- **并行方法**: 无
- **后置方法**: 执行 Pretotype → 分析结果 → 决策
