# Product Strategy (产品战略九段式)

## 目的

通过九个核心要素系统化构建产品战略，确保产品方向正确、执行有据可依、成果可衡量，避免「功能工厂」陷阱。

## 核心概念

```
产品战略九段式框架：

1. 愿景（Vision）
   └─ 我们要去哪里？产品最终要成为什么？

2. 目标客户（Target Customer）
   └─ 我们为谁服务？谁是核心用户？

3. 客户需求（Customer Needs）
   └─ 他们有什么问题和需求？

4. 竞争格局（Competitive Landscape）
   └─ 竞争对手是谁？我们如何差异化？

5. 价值主张（Value Proposition）
   └─ 我们提供什么独特价值？

6. 商业模式（Business Model）
   └─ 我们如何赚钱？

7. 成功指标（Success Metrics）
   └─ 我们如何衡量成功？

8. 增长策略（Growth Strategy）
   └─ 我们如何增长？

9. 防御性（Defensibility）
   └─ 我们如何保持竞争优势？
```

**核心思想**:
- 产品战略 ≠ 产品路线图
- 战略是「为什么做」，路线图是「做什么」
- 战略需要不断验证和调整

## 何时使用

- ✅ 新产品立项，定义产品方向
- ✅ 产品转型，重新思考产品定位
- ✅ 年度规划，制定产品战略
- ✅ 向团队/投资者阐述产品方向
- ✅ 产品团队对战略缺乏共识

## 使用步骤

### Step 1: 定义产品愿景（Vision）

**问题**: 产品最终要成为什么？我们想去哪里？

**愿景特征**:
- **方向性**：指明前进方向
- **激励性**：激励团队和用户
- **长期性**：5-10年甚至更长时间
- **情感性**：激发情感共鸣

**格式**:
```
产品愿景：

愿景陈述（一句话）：
[动词] + [目标用户] + [核心价值] + [最终状态]

示例：
- Airbnb: "Belong anywhere"（归属感）
- Spotify: "Give people access to all the music in the world"（音乐触手可及）
- LinkedIn: "Connect the world's professionals to make them more productive and successful"（连接全球专业人士）

我们的愿景：
[愿景陈述]

愿景阐述（一段话）：
[详细描述产品最终要实现的状态，为什么这个愿景重要，如何影响世界]

愿景关键词：
1. [关键词1]：[解释]
2. [关键词2]：[解释]
3. [关键词3]：[解释]
```

**检查点**:
- [ ] 愿景是否足够宏大？
- [ ] 愿景是否足够具体？
- [ ] 愿景是否能激励团队？
- [ ] 愿景是否与公司使命一致？

### Step 2: 定义目标客户（Target Customer）

**问题**: 我们为谁服务？谁是核心用户？

**客户分层**:
- **核心客户**：最重要、最核心的客户群
- **次要客户**：重要但非核心的客户群
- **边缘客户**：可能服务但不聚焦的客户群

**格式**:
```
目标客户：

核心客户：
1. [客户群1]
   - 人口特征：[年龄、性别、地域、收入]
   - 行为特征：[使用场景、消费习惯]
   - 心理特征：[价值观、生活方式]
   - 痛点强度：[高/中/低]
   - 付费意愿：[高/中/低]
   - 市场规模：[XX万人/XX亿元]

次要客户：
1. [客户群2]
   - ...

客户画像（Persona）：
[客户姓名]，[年龄]岁，[职业]
"一句话描述客户的状态和需求"

场景描述：
[客户]在[场景]下遇到[问题]，想要[解决方案]，
但现有方案[不足之处]，因此[痛点]。
```

**检查点**:
- [ ] 客户定义是否足够具体？
- [ ] 是否区分了核心和次要客户？
- [ ] 是否有客户画像和场景描述？
- [ ] 是否验证过客户存在？

### Step 3: 定义客户需求（Customer Needs）

**问题**: 客户有什么问题？他们真正需要什么？

**需求层次**:
- **痛点（Pain Points）**：客户遇到的问题和困扰
- **收益（Gains）**：客户期望的结果和好处
- **Jobs to be Done**：客户要完成的任务

**格式**:
```
客户需求：

痛点（Pain Points）：
1. [痛点1]
   - 具体描述：[场景+问题]
   - 严重程度：[高/中/低]
   - 发生频率：[高/中/低]
   - 客户占比：[XX%]

2. [痛点2]
   - ...

收益期望（Gains）：
1. [收益1]
   - 具体描述：[期望的结果]
   - 重要性：[高/中/低]
   - 客户占比：[XX%]

2. [收益2]
   - ...

Jobs to be Done：
1. [JTBD1]
   - 功能性任务：[具体要完成的任务]
   - 情感性任务：[任务背后的情感需求]
   - 社会性任务：[任务背后的社会需求]
```

**检查点**:
- [ ] 是否识别了客户的真实痛点？
- [ ] 是否区分了功能性和情感性需求？
- [ ] 是否有调研数据支撑？
- [ ] 是否排定了优先级？

### Step 4: 分析竞争格局（Competitive Landscape）

**问题**: 竞争对手是谁？我们如何差异化？

**竞争分析维度**:
- **直接竞争**：同类产品
- **间接竞争**：替代产品
- **潜在竞争**：可能进入的玩家

**格式**:
```
竞争格局：

直接竞争对手：
1. [竞争对手1]
   - 产品定位：[定位描述]
   - 核心优势：[优势描述]
   - 核心劣势：[劣势描述]
   - 市场份额：[XX%]
   - 用户评价：[用户反馈]

2. [竞争对手2]
   - ...

间接竞争对手：
1. [替代品1]
   - 替代方式：[如何替代]
   - 优劣势对比：[对比分析]

2. [替代品2]
   - ...

潜在竞争对手：
1. [潜在竞争者1]
   - 进入可能性：[高/中/低]
   - 进入方式：[方式描述]
   - 潜在威胁：[威胁描述]

竞争矩阵：
         功能丰富度
              │
    竞争对手A │   竞争对手B
              │
──────────────┼────────────── 价格
              │
    我们      │   竞争对手C
              │
         市场份额
```

**检查点**:
- [ ] 是否识别了所有竞争对手？
- [ ] 是否分析了竞争对手的优势劣势？
- [ ] 是否识别了差异化机会？

### Step 5: 定义价值主张（Value Proposition）

**问题**: 我们提供什么独特价值？为什么客户选择我们？

**价值主张维度**:
- **功能价值**：产品功能解决问题
- **情感价值**：产品带来的情感体验
- **社会价值**：产品带来的社会认同
- **经济价值**：产品带来的经济收益

**格式**:
```
价值主张：

核心价值主张（一句话）：
针对[目标客户]，我们提供[产品/服务]，
帮助他们[核心价值]，相比[竞争对手]，我们更[差异化优势]。

价值主张画布：

客户任务    │  我们的价值主张
─────────────┼────────────────────────
痛点         │  痛点缓解
1. ...      │  1. ...
2. ...      │  2. ...
            │
收益期望     │  收益创造
1. ...      │  1. ...
2. ...      │  2. ...

差异化优势：
1. [差异化点1]
   - 具体描述：[描述]
   - 可复制性：[难/中/易]
   - 客户感知度：[高/中/低]

2. [差异化点2]
   - ...
```

**检查点**:
- [ ] 价值主张是否清晰？
- [ ] 是否与客户痛点对应？
- [ ] 是否有明确的差异化？
- [ ] 是否可以用一句话说清？

### Step 6: 定义商业模式（Business Model）

**问题**: 我们如何赚钱？如何可持续增长？

**商业模式要素**:
- **收入模式**：订阅、交易、广告、授权...
- **定价策略**：免费、Freemium、分层定价...
- **成本结构**：固定成本、可变成本...

**格式**:
```
商业模式：

收入模式：
- 主营收入：[收入来源1] - 占比[XX%]
  - 定价模式：[一次性/订阅/按量...]
  - 价格区间：[X-Y元]
- 次营收入：[收入来源2] - 占比[XX%]
  - ...

定价策略：
- 免费版：[功能限制]
- 基础版：[价格] - [功能]
- 高级版：[价格] - [功能]
- 企业版：[价格] - [功能]

成本结构：
- 固定成本：[XX万/月]
  - 人力：[XX万]
  - 场地：[XX万]
  - 设备：[XX万]
- 可变成本：[按用户/按量]
  - 云服务：[XX元/用户]
  - 营销：[XX元/用户]

单位经济：
- 客户获取成本（CAC）：[XX元]
- 客户终身价值（LTV）：[XX元]
- LTV/CAC：[X倍]
- 回收周期：[X月]

盈利预测：
- 盈亏平衡点：[X年X月]
- 月度收入目标：[X万/月]
- 年度收入目标：[X万/年]
```

**检查点**:
- [ ] 收入模式是否清晰？
- [ ] 单位经济是否健康？
- [ ] 成本结构是否合理？
- [ ] 是否有盈利路径？

### Step 7: 定义成功指标（Success Metrics）

**问题**: 我们如何衡量成功？如何知道我们在进步？

**指标层次**:
- **北极星指标**：最重要的单一指标
- **输入指标**：可以控制的指标
- **输出指标**：需要观察的指标
- **健康指标**：确保不牺牲长期价值

**格式**:
```
成功指标：

北极星指标：
[指标名称]：[定义] - 当前值：[X] - 目标值：[Y]

关键指标（按阶段）：
获取阶段：
- 新增用户数：[当前值] → [目标值]
- 渠道转化率：[当前值] → [目标值]
- 获客成本（CAC）：[当前值] → [目标值]

激活阶段：
- 注册转化率：[当前值] → [目标值]
- 核心功能使用率：[当前值] → [目标值]
- 首次体验完成率：[当前值] → [目标值]

留存阶段：
- 次日留存：[当前值] → [目标值]
- 7日留存：[当前值] → [目标值]
- 30日留存：[当前值] → [目标值]

收入阶段：
- 付费转化率：[当前值] → [目标值]
- ARPU：[当前值] → [目标值]
- LTV：[当前值] → [目标值]

推荐阶段：
- NPS：[当前值] → [目标值]
- 邀请率：[当前值] → [目标值]
- 病毒系数：[当前值] → [目标值]

健康指标：
- 用户满意度：[当前值] → [目标值]
- 系统稳定性：[当前值] → [目标值]
- 团队满意度：[当前值] → [目标值]

指标仪表盘：
[描述如何追踪和展示这些指标]
```

**检查点**:
- [ ] 是否有明确的北极星指标？
- [ ] 指标是否可衡量、可追踪？
- [ ] 是否区分了输入和输出指标？
- [ ] 是否有健康指标防止短视？

### Step 8: 定义增长策略（Growth Strategy）

**问题**: 我们如何增长？如何获取更多用户？

**增长策略维度**:
- **用户增长**：如何获取更多用户
- **收入增长**：如何提升收入
- **市场增长**：如何拓展市场

**格式**:
```
增长策略：

用户增长策略：
1. [策略1]：[策略描述]
   - 目标：[量化目标]
   - 方法：[具体做法]
   - 渠道：[获客渠道]
   - 时间：[起止时间]
   - 预算：[预算需求]

2. [策略2]：...

收入增长策略：
1. [策略1]：[策略描述]
   - 方法：[具体做法]
   - 预期效果：[量化效果]

2. [策略2]：...

市场拓展策略：
1. [市场1]：[市场描述]
   - 进入方式：[自建/合作/收购]
   - 时间规划：[起止时间]

2. [市场2]：...

增长模型：
获客 → 激活 → 留存 → 收入 → 推荐
[描述每个环节的增长策略和转化路径]
```

**检查点**:
- [ ] 增长策略是否具体？
- [ ] 是否有明确的增长渠道？
- [ ] 是否有量化目标？
- [ ] 是否考虑了增长成本？

### Step 9: 定义防御性（Defensibility）

**问题**: 我们如何保持竞争优势？如何防止被复制？

**防御性来源**:
- **网络效应**：用户越多价值越大
- **规模效应**：规模越大成本越低
- **品牌效应**：品牌信任度和认知度
- **技术壁垒**：专利、算法、技术积累
- **数据壁垒**：独家数据、数据网络效应
- **渠道壁垒**：独家渠道、渠道垄断
- **转换成本**：用户迁移成本

**格式**:
```
防御性：

核心护城河：
1. [护城河1]
   - 具体描述：[如何形成]
   - 可复制性：[难/中/易]
   - 持续时间：[短期/中期/长期]
   - 强化方法：[如何加强]

2. [护城河2]
   - ...

防御策略：
1. [策略1]：[策略描述]
   - 目标：[强化哪个护城河]
   - 方法：[具体做法]
   - 时间规划：[起止时间]

2. [策略2]：...

潜在威胁：
1. [威胁1]
   - 威胁描述：[描述]
   - 可能性：[高/中/低]
   - 应对策略：[如何应对]

2. [威胁2]
   - ...

竞争优势演进路径：
[描述护城河如何随时间演化和加强]
```

**检查点**:
- [ ] 是否有真正的护城河？
- [ ] 护城河是否可持续？
- [ ] 是否有强化护城河的计划？
- [ ] 是否识别了潜在威胁？

## 常见错误

- ❌ 愿景过于宽泛，缺乏方向性
- ❌ 目标客户定义不清，试图服务所有人
- ❌ 客户需求主观臆断，缺乏调研
- ❌ 价值主张缺乏差异化
- ❌ 商业模式不清晰，没有盈利路径
- ❌ 成功指标过多，没有北极星指标
- ❌ 增长策略过于宽泛，缺乏可执行性
- ❌ 没有防御性，容易被复制
- ❌ 战略制定后不验证，束之高阁
