# Product Vision (产品愿景)

## 目的

定义产品最终要实现的状态，为团队提供长期方向指引、情感激励和决策依据，确保团队朝着同一个方向努力。

## 核心概念

```
产品愿景层次：

┌─────────────────────────────────────────────┐
│           公司使命 (Mission)                │
│     公司存在的目的，永不改变                 │
├─────────────────────────────────────────────┤
│           产品愿景 (Vision)                 │
│     产品最终要成为什么，5-10年目标           │
├─────────────────────────────────────────────┤
│           产品战略 (Strategy)               │
│     如何实现愿景，3年规划                    │
├─────────────────────────────────────────────┤
│           产品路线图 (Roadmap)              │
│     具体要做什么，年度/季度规划               │
├─────────────────────────────────────────────┤
│           产品目标 (Goals)                  │
│     可衡量的短期目标，OKR                    │
└─────────────────────────────────────────────┘
```

**愿景的特征**:
- **方向性**：指明前进方向，不轻易改变
- **激励性**：激发团队热情和用户共鸣
- **长期性**：5-10年甚至更长时间
- **可达性**：有实现的可能性，不是幻想
- **清晰性**：容易理解和传播

## 何时使用

- ✅ 新产品立项，定义产品方向
- ✅ 产品转型，重新思考产品定位
- ✅ 团队对产品方向缺乏共识
- ✅ 向投资者/合作伙伴展示产品愿景
-  年度规划，检验愿景是否仍适用
- ✅ 新团队组建，统一方向认知

## 使用步骤

### Step 1: 收集愿景输入

**问题**: 我们需要什么信息来构建愿景？

**输入来源**:
- **公司使命**：公司存在的目的
- **用户需求**：用户最深层的渴望
- **市场趋势**：行业和技术发展趋势
- **团队能力**：团队的核心能力
- **价值观**：团队共同认可的价值

**格式**:
```
愿景输入：

公司使命：
[公司使命陈述]

用户深层需求：
1. [需求1]：[用户真正想要的是什么？]
2. [需求2]：[用户最深的渴望是什么？]

市场趋势：
1. [趋势1]：[未来5-10年市场会如何变化？]
2. [趋势2]：[技术会如何发展？]

团队能力：
1. [能力1]：[团队最擅长什么？]
2. [能力2]：[团队有什么独特优势？]

核心价值观：
1. [价值观1]：[团队最看重什么？]
2. [价值观2]：[团队坚持什么原则？]

灵感来源：
- 竞品愿景：[竞争对手的愿景是什么？]
- 行业标杆：[行业标杆的愿景是什么？]
- 其他启发：[有什么启发性的愿景？]
```

**检查点**:
- [ ] 是否理解了公司使命？
- [ ] 是否挖掘了用户深层需求？
- [ ] 是否分析了市场趋势？
- [ ] 是否识别了团队能力？

### Step 2: 愿景头脑风暴

**问题**: 产品最终要成为什么？

**方法**:
- **未来新闻**：想象5-10年后的新闻头条
- **用户见证**：想象用户如何描述产品改变了他们的生活
- **团队对话**：团队共同探讨产品方向
- **逆向工程**：从成功产品反推愿景

**格式**:
```
愿景头脑风暴：

未来新闻（5-10年后）：
标题：[新闻标题]
正文：[新闻内容，描述产品的影响和成就]

用户见证：
"在我最需要帮助的时候，[产品]给了我支持，改变了我的生活。
现在我无法想象没有它的日子。" —— [用户姓名]

团队对话：
- 团队成员1：[你认为产品应该成为什么？]
- 团队成员2：[你认为产品应该成为什么？]
- ...

逆向工程：
如果我们成功了，那时候：
- 用户如何描述我们？[描述]
- 竞争对手如何看我们？[描述]
- 行业如何被我们改变？[描述]
- 我们改变了什么？[描述]

愿景关键词：
1. [关键词1]
2. [关键词2]
3. [关键词3]
```

**检查点**:
- [ ] 是否激发了想象力？
- [ ] 是否让团队参与进来？
- [ ] 是否考虑了多个视角？

### Step 3: 起草愿景陈述

**问题**: 如何用一句话表达愿景？

**愿景陈述结构**:
- **动词**：连接/赋能/让/帮助/使...
- **目标用户**：为谁
- **核心价值**：提供什么价值
- **最终状态**：达到什么状态

**格式**:
```
愿景陈述草稿：

草稿1：
[动词] + [目标用户] + [核心价值] + [最终状态]

草稿2：
[动词] + [目标用户] + [核心价值] + [最终状态]

草稿3：
[动词] + [目标用户] + [核心价值] + [最终状态]

愿景陈述示例：
- Airbnb: "Belong anywhere"（归属感）
- Spotify: "Give people access to all the music in the world"（音乐触手可及）
- LinkedIn: "Connect the world's professionals to make them more productive and successful"（连接全球专业人士）
- Google: "Organize the world's information and make it universally accessible and useful"（组织全球信息）
- Tesla: "Accelerate the world's transition to sustainable energy"（加速世界向可持续能源转型）

我们的愿景陈述：
[选择或修改一个草稿]
```

**检查点**:
- [ ] 是否简洁（一句话）？
- [ ] 是否清晰易懂？
- [ ] 是否足够宏大？
- [ ] 是否有情感共鸣？

### Step 4: 验证愿景

**问题**: 这个愿景是否有效？

**验证维度**:
- **方向性**：是否指明了方向？
- **激励性**：是否激发了热情？
- **长期性**：是否可以持续5-10年？
- **可达性**：是否有可能实现？
- **清晰性**：是否容易理解？
- **差异化**：是否区别于竞争对手？
- **一致性**：是否与公司使命一致？

**格式**:
```
愿景验证：

内部验证（团队）：
- 团队成员1：[你觉得这个愿景如何？是否激励你？]
- 团队成员2：[你觉得这个愿景如何？是否激励你？]
- ...

外部验证（用户）：
- 用户1：[这个愿景是否打动你？为什么？]
- 用户2：[这个愿景是否打动你？为什么？]
- ...

专家验证（顾问/投资者）：
- 专家1：[这个愿景是否可行？有什么建议？]
- 专家2：[这个愿景是否可行？有什么建议？]
- ...

验证评分：
| 维度 | 评分(1-10) | 反馈 |
|-----|----------|------|
| 方向性 | ... | ... |
| 激励性 | ... | ... |
| 长期性 | ... | ... |
| 可达性 | ... | ... |
| 清晰性 | ... | ... |
| 差异化 | ... | ... |
| 一致性 | ... | ... |
| 总分 | ... | ... |
```

**检查点**:
- [ ] 是否收集了多方反馈？
- [ ] 是否有量化评分？
- [ ] 是否根据反馈修改？

### Step 5: 愿景阐述

**问题**: 如何让愿景更丰满、更有说服力？

**阐述要素**:
- **为什么**：为什么这个愿景重要
- **为谁**：为谁创造价值
- **如何实现**：大致方向
- **最终状态**：成功时的样子

**格式**:
```
愿景阐述：

愿景陈述：
[一句话愿景]

愿景故事（为什么这个愿景重要）：
[2-3段话阐述愿景的背景、意义和价值]

用户价值（为谁创造价值）：
[描述用户如何从产品中受益]

实现路径（如何实现）：
[大致方向和策略，不需要具体细节]

成功景象（最终状态）：
[描述当愿景实现时的样子]

示例：
愿景陈述："让每个人都能轻松创作"

愿景故事：
创作是人类最基本的需求之一。然而，传统创作工具门槛高、学习成本大，
让很多人望而却步。我们相信，每个人都有创作的潜力，只是缺少合适的工具。
我们的愿景是让创作变得像呼吸一样自然，让每个人都能轻松表达自己的想法。

用户价值：
无论是设计师、内容创作者，还是普通用户，都能用我们的工具快速创作出
专业级别的作品，表达自己的创意，分享自己的故事。

实现路径：
通过AI技术降低创作门槛，通过模板化简化创作流程，通过社区激励创作分享。

成功景象：
当我们的愿景实现时，每个人都能随时随地创作，创作不再是少数人的专利，
而是每个人的日常。
```

**检查点**:
- [ ] 是否有情感共鸣？
- [ ] 是否解释了为什么？
- [ ] 是否描述了最终状态？

### Step 6: 愿景传播

**问题**: 如何让愿景深入人心？

**传播对象**:
- **团队**：内部成员
- **用户**：潜在和现有用户
- **投资者**：潜在和现有投资者
- **合作伙伴**：供应商、渠道等
- **公众**：媒体、公众

**格式**:
```
愿景传播：

团队传播：
- 愿景分享会：[时间、形式]
- 团队Workshop：[时间、形式]
- 日常提醒：[如何日常提醒]
- 招聘融入：[如何在招聘中传递]

用户传播：
- 产品介绍页：[如何呈现]
- 品牌传播：[如何传递]
- 用户故事：[如何讲述]

投资者传播：
- 融资材料：[如何呈现]
- 投资者沟通：[如何讲述]

合作伙伴传播：
- 合作洽谈：[如何传递]
- 合作材料：[如何呈现]

公众传播：
- 媒体报道：[如何讲述]
- 公众演讲：[如何传递]

愿景宣言（简短版）：
[用于传播的简短版本，适合口头表达和社交媒体]
```

**检查点**:
- [ ] 是否有传播计划？
- [ ] 是否针对不同受众调整？
- [ ] 是否有简短版本？

### Step 7: 愿景落地

**问题**: 如何让愿景指导日常工作？

**落地机制**:
- **战略连接**：愿景 → 战略 → 路线图 → 目标
- **决策依据**：用愿景检验决策
- **定期回顾**：定期检查进展
- **庆祝里程碑**：庆祝向愿景迈进的成就

**格式**:
```
愿景落地：

战略连接：
- 产品战略：[如何与愿景连接]
- 产品路线图：[如何体现愿景]
- 产品目标（OKR）：[如何与愿景对齐]

决策依据：
- 决策问题：[当前要做的决策]
- 愿景检验：[这个决策是否让我们更接近愿景？]
- 决策结果：[如何决策]

定期回顾：
- 回顾频率：[季度/半年/年度]
- 回顾内容：[我们离愿景还有多远？是否偏离方向？]
- 调整机制：[何时需要调整愿景？]

庆祝里程碑：
- 里程碑1：[成就描述] - [庆祝方式]
- 里程碑2：[成就描述] - [庆祝方式]

团队仪式：
- 愿景墙：[在办公场所展示愿景]
- 会议开场：[重要会议前重申愿景]
- 新人培训：[向新人传递愿景]
```

**检查点**:
- [ ] 是否与战略连接？
- [ ] 是否有决策机制？
- [ ] 是否有回顾机制？
- [ ] 是否有庆祝仪式？

## 常见错误

- ❌ 愿景过于宽泛，缺乏方向性（"我们要成为最好的公司"）
- ❌ 愿景过于具体，变成了目标（"我们要达到1000万用户"）
- ❌ 愿景过于保守，缺乏激励性
- ❌ 愿景过于理想，缺乏可达性
- ❌ 愿景缺乏情感，无法激发共鸣
- ❌ 愿景与公司使命不一致
- ❌ 愿景制定后束之高阁，不传播、不落地
- ❌ 愿景频繁改变，团队无所适从

## 与其他方法的关系

- **前置方法**: company-mission（公司使命）、user-research（用户研究）
- **后置方法**: product-strategy（产品战略）、product-roadmap（产品路线图）
- **并行方法**: value-proposition（价值主张）、product-strategy（产品战略九段式）
