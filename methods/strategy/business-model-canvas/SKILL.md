# Business Model Canvas (商业模式画布)

## 目的

通过一张画布系统化呈现商业模式的9个核心模块，帮助团队统一语言、发现缺失环节、快速迭代商业模式假设。

## 核心概念

```
┌─────────────┬─────────────┬─────────────┬─────────────┬─────────────┐
│   关键合作   │   关键活动   │   价值主张   │   客户关系   │   目标客户   │
│ Key Partners│Key Activities│Value Prop  │Customer Rel.│Segments    │
│             │             │             │             │             │
│             ├─────────────┤             ├─────────────┤             │
│             │   关键资源   │             │   渠道通路   │             │
│             │Key Resources│             │ Channels    │             │
├─────────────┴─────────────┴─────────────┴─────────────┴─────────────┤
│                        成本结构 Cost Structure                        │
├─────────────────────────────────────────────────────────────────────┤
│                        收入来源 Revenue Streams                       │
└─────────────────────────────────────────────────────────────────────┘
```

**右侧（前台）**：面向市场 - 客户、渠道、关系、收入
**左侧（后台）**：支撑系统 - 合作伙伴、活动、资源、成本
**中间**：价值主张 - 连接前台后台的核心

## 何时使用

- ✅ 新产品/新业务立项，需要定义商业模式
- ✅ 现有业务模式需要转型或优化
- ✅ 向投资者/合作伙伴解释商业模式
- ✅ 团队对商业模式缺乏共识
- ✅ 竞品分析，理解对手商业模式

## 使用步骤

### Step 1: 定义客户细分（Customer Segments）

**问题**: 我们为谁创造价值？谁是我们最重要的客户？

**维度**:
- 大众市场（Mass Market）
- 小众市场（Niche Market）
- 区隔市场（Segmented）
- 多边平台（Diversified）

**格式**:
```
客户细分：
1. [主要客户群]
   - 人口特征：[年龄、性别、地域、收入]
   - 行为特征：[使用场景、消费习惯]
   - 痛点：[核心痛点1, 核心痛点2]

2. [次要客户群]
   - ...
```

**检查点**:
- [ ] 客户群是否足够具体？
- [ ] 是否区分了付费用户和使用用户？
- [ ] 是否识别了核心客户群？

### Step 2: 定义价值主张（Value Propositions）

**问题**: 我们帮助客户解决了什么问题？满足了什么需求？提供了什么价值？

**维度**:
- 新颖性（Newness）
- 性能（Performance）
- 定制化（Customization）
- 设计（Design）
- 品牌/地位（Brand/Status）
- 价格（Price）
- 成本削减（Cost Reduction）
- 风险削减（Risk Reduction）
- 可达性（Accessibility）
- 便利性/可用性（Convenience/Usability）

**格式**:
```
价值主张：
针对 [客户群]，我们提供 [产品/服务]，
帮助他们 [解决的问题/满足的需求]，
相比 [竞品/替代方案]，我们更 [差异化优势]。
```

**检查点**:
- [ ] 是否与客户痛点直接对应？
- [ ] 是否有明确的差异化？
- [ ] 是否可以用一句话概括？

### Step 3: 定义渠道通路（Channels）

**问题**: 我们通过什么渠道触达客户？客户如何了解、评估、购买、使用我们的产品？

**阶段**:
1. 认知（Awareness）
2. 评估（Evaluation）
3. 购买（Purchase）
4. 交付（Delivery）
5. 售后（After Sales）

**格式**:
```
渠道通路：
- 认知渠道：[App Store、社交媒体、口碑推荐...]
- 转化渠道：[官网、应用商店、线下门店...]
- 交付渠道：[线上下载、线下配送...]
- 售后渠道：[客服热线、在线客服、社区...]
```

### Step 4: 定义客户关系（Customer Relationships）

**问题**: 我们与客户建立什么类型的关系？如何获取、留存和增长客户？

**类型**:
- 个人协助（Personal Assistance）
- 专属个人助理（Dedicated Personal Assistance）
- 自助服务（Self-Service）
- 自动化服务（Automated Services）
- 社区（Communities）
- 共创（Co-creation）

**格式**:
```
客户关系：
- 获取：[免费试用、内容营销、推荐激励...]
- 留存：[会员体系、社区运营、个性化推荐...]
- 增长：[病毒传播、交叉销售、向上销售...]
```

### Step 5: 定义收入来源（Revenue Streams）

**问题**: 客户为什么价值付费？我们现在如何赚钱？未来如何赚钱？

**类型**:
- 资产销售（Asset Sale）
- 使用费（Usage Fee）
- 订阅费（Subscription Fees）
- 租赁/出租（Lending/Renting/Leasing）
- 授权许可（Licensing）
- 佣金（Brokerage Fees）
- 广告（Advertising）

**格式**:
```
收入来源：
1. [收入类型1]
   - 定价模式：[一次性/订阅/按量...]
   - 价格区间：[X-Y元]
   - 占比：[XX%]

2. [收入类型2]
   - ...
```

### Step 6: 定义关键资源（Key Resources）

**问题**: 我们的价值主张需要什么资源？渠道需要什么资源？客户关系需要什么资源？收入来源需要什么资源？

**类型**:
- 实物资源（Physical）
- 知识产权（Intellectual）
- 人力资源（Human）
- 金融资源（Financial）

**格式**:
```
关键资源：
- 实物：[服务器、办公场所、设备...]
- 知识产权：[专利、品牌、数据、算法...]
- 人力：[核心团队、专家顾问...]
- 金融：[现金流、融资能力...]
```

### Step 7: 定义关键活动（Key Activities）

**问题**: 我们的价值主张需要做什么？渠道需要做什么？客户关系需要做什么？收入来源需要做什么？

**类型**:
- 生产（Production）
- 问题解决（Problem Solving）
- 平台/网络（Platform/Network）

**格式**:
```
关键活动：
- 产品开发：[需求分析、设计、开发、测试...]
- 运营维护：[内容运营、用户运营、客服...]
- 市场营销：[品牌推广、渠道拓展、活动策划...]
- 商务合作：[BD谈判、合同签署、关系维护...]
```

### Step 8: 定义关键合作（Key Partnerships）

**问题**: 谁是我们的关键合作伙伴？我们从合作伙伴那里获取什么资源？合作伙伴执行什么活动？

**动机**:
- 优化与规模经济（Optimization and Economy of Scale）
- 降低风险与不确定性（Reduction of Risk and Uncertainty）
- 获取特定资源与活动（Acquisition of Particular Resources and Activities）

**格式**:
```
关键合作：
1. [合作伙伴类型1]
   - 合作方：[具体名称]
   - 合作内容：[提供的资源/活动]
   - 合作模式：[战略合作、供应商、分销商...]

2. [合作伙伴类型2]
   - ...
```

### Step 9: 定义成本结构（Cost Structure）

**问题**: 关键资源花费多少？关键活动花费多少？

**类型**:
- 成本驱动（Cost-Driven）
- 价值驱动（Value-Driven）

**要素**:
- 固定成本（Fixed Costs）
- 可变成本（Variable Costs）
- 规模经济（Economies of Scale）
- 范围经济（Economies of Scope）

**格式**:
```
成本结构：
固定成本：
- 人力成本：[X万/月]
- 场地租金：[Y万/月]
- 设备折旧：[Z万/月]

可变成本：
- 云服务器：[按量计费]
- 营销费用：[按转化计费]
- 客服成本：[按工单计费]

总成本预估：[X万/月]
```

## 常见错误

- ❌ 价值主张过于宽泛，缺乏差异化
- ❌ 客户细分不够具体，无法定位
- ❌ 混淆渠道和客户关系
- ❌ 忽视免费用户和付费用户的区别
- ❌ 成本结构遗漏隐性成本
- ❌ 收入来源单一，缺乏多元化设计
- ❌ 关键合作描述不清合作动机
- ❌ 各模块之间缺乏一致性检验
