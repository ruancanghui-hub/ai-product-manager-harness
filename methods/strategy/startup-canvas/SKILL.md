# Startup Canvas (创业画布)

## 目的

为创业项目提供一张完整的商业计划画布，结合产品战略（愿景、客户、价值主张）和商业模式（收入、成本、增长），帮助创业者系统化思考创业项目。

## 核心概念

```
创业画布结构：

┌─────────────────────────────────────────────────────────────────┐
│                      产品战略（上半部分）                        │
├─────────────┬─────────────┬─────────────┬─────────────────────┤
│   产品愿景   │  目标客户   │  客户需求   │   价值主张           │
│   Vision    │  Customer   │   Needs     │  Value Prop         │
│             │             │             │                      │
│  我们要去哪  │  为谁服务   │  什么需求   │   提供什么价值        │
├─────────────┼─────────────┼─────────────┼─────────────────────┤
│  竞争格局    │  商业模式   │  成功指标   │   增长策略           │
│ Competition │ Business    │  Metrics    │  Growth             │
│             │  Model      │             │                      │
│  如何差异化  │  如何赚钱   │  如何衡量   │   如何增长           │
├─────────────┴─────────────┴─────────────┴─────────────────────┤
│                      商业模式（下半部分）                        │
├─────────────────────────────────────────────────────────────────┤
│                       成本结构 Cost Structure                    │
├─────────────────────────────────────────────────────────────────┤
│                       收入来源 Revenue Streams                   │
├─────────────────────────────────────────────────────────────────┤
│                       防御性 Defensibility                       │
└─────────────────────────────────────────────────────────────────┘
```

**核心思想**:
- 上半部分：产品战略 - 做正确的事
- 下半部分：商业模式 - 可持续发展
- 整体：从愿景到防御的完整逻辑链

## 何时使用

- ✅ 创业项目立项
- ✅ 向投资者展示创业计划
- ✅ 创业团队统一认知
- ✅ 创业项目复盘和调整
- ✅ 创业竞赛和路演准备

## 使用步骤

### Step 1: 定义产品愿景（Vision）

**问题**: 产品最终要成为什么？

**格式**:
```
产品愿景：

愿景陈述（一句话）：
[动词] + [目标用户] + [核心价值] + [最终状态]

愿景阐述：
[2-3段话阐述愿景的背景、意义和价值]

愿景关键词：
1. [关键词1]：[解释]
2. [关键词2]：[解释]
3. [关键词3]：[解释]

时间范围：
- 短期目标（1年）：[目标]
- 中期目标（3年）：[目标]
- 长期愿景（5-10年）：[愿景]
```

**检查点**:
- [ ] 愿景是否足够宏大？
- [ ] 愿景是否足够具体？
- [ ] 愿景是否能激励团队？

### Step 2: 定义目标客户（Target Customer）

**问题**: 我们为谁服务？

**格式**:
```
目标客户：

核心客户：
1. [客户群1]
   - 人口特征：[年龄、性别、地域、收入]
   - 行为特征：[使用场景、消费习惯]
   - 心理特征：[价值观、生活方式]
   - 痛点强度：[高/中/低]
   - 市场规模：[XX万人/XX亿元]

早期采用者：
[具体描述早期采用者是谁、在哪里、如何找到他们]

客户画像（Persona）：
[客户姓名]，[年龄]岁，[职业]
"一句话描述客户的状态和需求"
```

**检查点**:
- [ ] 客户定义是否足够具体？
- [ ] 是否识别了早期采用者？
- [ ] 是否有市场规模估算？

### Step 3: 定义客户需求（Customer Needs）

**问题**: 客户有什么问题和需求？

**格式**:
```
客户需求：

痛点（Pain Points）：
1. [痛点1] - 严重程度：[高] - 频率：[高]
2. [痛点2] - 严重程度：[高] - 频率：[中]
3. [痛点3] - 严重程度：[中] - 频率：[高]

收益期望（Gains）：
1. [收益1] - 重要性：[高]
2. [收益2] - 重要性：[高]
3. [收益3] - 重要性：[中]

Jobs to be Done：
"当[情境]时，我需要[完成任务]，因为我想要[动机]"

需求验证：
- 验证方法：[访谈/问卷/数据分析]
- 验证结果：[结果描述]
```

**检查点**:
- [ ] 是否识别了核心痛点？
- [ ] 是否有需求验证？
- [ ] 是否用JTBD描述需求？

### Step 4: 定义价值主张（Value Proposition）

**问题**: 我们提供什么独特价值？

**格式**:
```
价值主张：

核心价值主张（一句话）：
针对[目标客户]，我们提供[产品/服务]，
帮助他们[核心价值]，相比[替代方案]，我们更[差异化优势]。

差异化优势：
1. [差异化点1] - 可复制性：[难]
2. [差异化点2] - 可复制性：[中]
3. [差异化点3] - 可复制性：[易]

价值主张画布：
- 痛点缓解：[我们如何缓解痛点]
- 收益创造：[我们如何创造收益]
```

**检查点**:
- [ ] 价值主张是否清晰？
- [ ] 是否有明确的差异化？
- [ ] 是否与客户需求对应？

### Step 5: 分析竞争格局（Competitive Landscape）

**问题**: 竞争对手是谁？如何差异化？

**格式**:
```
竞争格局：

直接竞争对手：
1. [竞争对手1]
   - 产品定位：[定位]
   - 市场份额：[XX%]
   - 优势：[优势]
   - 劣势：[劣势]

2. [竞争对手2]
   - ...

间接竞争对手：
1. [替代品1] - 替代方式：[描述]

潜在竞争对手：
1. [潜在竞争者1] - 进入可能性：[高/中/低]

竞争策略：
- 差异化策略：[策略描述]
- 聚焦策略：[策略描述]
```

**检查点**:
- [ ] 是否识别了所有竞争对手？
- [ ] 是否有竞争策略？
- [ ] 是否分析了差异化机会？

### Step 6: 定义商业模式（Business Model）

**问题**: 如何赚钱？

**格式**:
```
商业模式：

收入模式：
- 主营收入：[收入来源] - 占比[XX%]
  - 定价模式：[订阅/交易/广告...]
  - 价格区间：[X-Y元]
- 次营收入：[收入来源] - 占比[XX%]

定价策略：
- 免费版：[功能限制]
- 基础版：[价格] - [功能]
- 高级版：[价格] - [功能]
- 企业版：[价格] - [功能]

渠道通路：
- 获客渠道：[渠道列表]
- 分销渠道：[渠道列表]

关键合作：
1. [合作伙伴1]：[合作内容]
2. [合作伙伴2]：[合作内容]
```

**检查点**:
- [ ] 收入模式是否清晰？
- [ ] 是否有定价策略？
- [ ] 是否有获客渠道？

### Step 7: 定义成功指标（Success Metrics）

**问题**: 如何衡量成功？

**格式**:
```
成功指标：

北极星指标：
[指标名称]：当前值[X] → 目标值[Y]

关键指标（AARRR）：
- 获取：新增用户数[X] → 目标[Y]
- 激活：注册转化率[X%] → 目标[Y%]
- 留存：7日留存率[X%] → 目标[Y%]
- 收入：付费转化率[X%] → 目标[Y%]
- 推荐：NPS[X] → 目标[Y]

里程碑：
- 3个月：[里程碑]
- 6个月：[里程碑]
- 12个月：[里程碑]
```

**检查点**:
- [ ] 是否有北极星指标？
- [ ] 是否有AARRR指标？
- [ ] 是否有里程碑？

### Step 8: 定义增长策略（Growth Strategy）

**问题**: 如何增长？

**格式**:
```
增长策略：

用户增长策略：
1. [策略1]：[具体做法] - 预期效果：[效果]
2. [策略2]：[具体做法] - 预期效果：[效果]

收入增长策略：
1. [策略1]：[具体做法] - 预期效果：[效果]
2. [策略2]：[具体做法] - 预期效果：[效果]

市场拓展策略：
1. [市场1]：[进入方式] - 时间：[时间]
2. [市场2]：[进入方式] - 时间：[时间]

增长模型：
获客 → 激活 → 留存 → 收入 → 推荐
[描述增长路径和关键转化]
```

**检查点**:
- [ ] 是否有用户增长策略？
- [ ] 是否有收入增长策略？
- [ ] 是否有增长模型？

### Step 9: 定义成本结构（Cost Structure）

**问题**: 成本结构如何？

**格式**:
```
成本结构：

一次性成本：
- 启动成本：[X万元]
- 设备投资：[Y万元]

月固定成本：
- 人力：[人数] × [平均薪资] = [总额]
- 场地：[金额]
- 云服务：[金额]
- 其他：[金额]
- 小计：[总额]

月可变成本：
- 营销费用：[按用户计费]
- 云服务器：[按量计费]
- 其他：[金额]

单位经济：
- CAC：[X元]
- LTV：[Y元]
- LTV/CAC：[Z倍]
- 回收周期：[W月]

烧钱率：[X万/月]
runway：[Y个月]
```

**检查点**:
- [ ] 是否有完整的成本结构？
- [ ] 是否计算了单位经济？
- [ ] 是否计算了runway？

### Step 10: 定义收入来源（Revenue Streams）

**问题**: 预期收入如何？

**格式**:
```
收入来源：

收入预测：
| 时间 | 用户数 | 付费用户 | ARPU | 月收入 | 年收入 |
|-----|--------|---------|------|--------|--------|
| 当前 | ... | ... | ... | ... | ... |
| 3个月 | ... | ... | ... | ... | ... |
| 6个月 | ... | ... | ... | ... | ... |
| 12个月 | ... | ... | ... | ... | ... |

盈亏平衡：
- 盈亏平衡点：[X万用户或Y万元收入]
- 预计达到时间：[时间]

融资计划：
- 融资需求：[X万元]
- 资金用途：[用途分配]
- 预期runway：[Y个月]
```

**检查点**:
- [ ] 是否有收入预测？
- [ ] 是否计算了盈亏平衡点？
- [ ] 是否有融资计划？

### Step 11: 定义防御性（Defensibility）

**问题**: 如何保持竞争优势？

**格式**:
```
防御性：

核心护城河：
1. [护城河1]
   - 具体描述：[如何形成]
   - 可复制性：[难/中/易]
   - 持续时间：[短期/中期/长期]

2. [护城河2]
   - ...

防御策略：
1. [策略1]：[如何强化护城河]
2. [策略2]：[如何强化护城河]

潜在威胁：
1. [威胁1] - 应对策略：[策略]
2. [威胁2] - 应对策略：[策略]
```

**检查点**:
- [ ] 是否有真正的护城河？
- [ ] 是否可持续？
- [ ] 是否识别了潜在威胁？

### Step 12: 风险评估与验证计划

**问题**: 有什么风险？如何验证？

**格式**:
```
风险评估：

关键风险：
1. [风险1]
   - 风险描述：[描述]
   - 可能性：[高/中/低]
   - 影响程度：[高/中/低]
   - 应对策略：[策略]

2. [风险2]
   - ...

验证计划：

关键假设：
1. [假设1]：[假设描述]
2. [假设2]：[假设描述]
3. [假设3]：[假设描述]

验证方法：
| 假设 | 验证方法 | 成功标准 | 时间 | 优先级 |
|-----|---------|---------|------|--------|
| ... | ... | ... | ... | ... |
```

**检查点**:
- [ ] 是否识别了关键风险？
- [ ] 是否识别了关键假设？
- [ ] 是否有验证计划？

## 常见错误

- ❌ 愿景过于宽泛，缺乏方向性
- ❌ 目标客户定义不清
- ❌ 客户需求主观臆断，缺乏验证
- ❌ 价值主张缺乏差异化
- ❌ 商业模式不清晰
- ❌ 成功指标过多，没有北极星指标
- ❌ 成本结构遗漏隐性成本
- ❌ 没有防御性，容易被复制
- ❌ 只做规划，不验证假设
