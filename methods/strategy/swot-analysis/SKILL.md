# SWOT Analysis (SWOT分析)

## 目的

系统化评估企业的内部能力（优势Strengths、劣势Weaknesses）和外部环境（机会Opportunities、威胁Threats），识别战略方向，制定行动计划。

## 核心概念

```
                内部因素                 外部因素
              (可控因素)              (不可控因素)
                    │                         │
        ┌───────────┴───────────┐   ┌──────────┴──────────┐
        │                       │   │                     │
        │     优势 (S)          │   │    机会 (O)         │
        │     Strengths         │   │    Opportunities    │
        │                       │   │                     │
积极影响 │  - 核心能力           │   │  - 市场趋势         │
        │  - 资源优势           │   │  - 技术变革         │
        │  - 品牌资产           │   │  - 政策利好         │
        │                       │   │                     │
        ├───────────────────────┤   ├─────────────────────┤
        │                       │   │                     │
        │     劣势 (W)          │   │    威胁 (T)         │
        │     Weaknesses        │   │    Threats          │
        │                       │   │                     │
消极影响 │  - 能力短板           │   │  - 竞争加剧         │
        │  - 资源不足           │   │  - 市场萎缩         │
        │  - 流程问题           │   │  - 政策限制         │
        │                       │   │                     │
        └───────────────────────┘   └─────────────────────┘
```

**交叉策略**:
- **SO策略**：利用优势抓住机会（进攻型）
- **WO策略**：克服劣势抓住机会（补短板型）
- **ST策略**：利用优势应对威胁（防御型）
- **WT策略**：减少劣势避免威胁（生存型）

## 何时使用

- ✅ 战略规划阶段，需要系统评估现状
- ✅ 新产品立项，分析可行性
- ✅ 竞争态势分析，制定应对策略
- ✅ 组织变革前，识别风险和机会
- ✅ 个人职业规划，分析自身优势劣势
- ✅ 投资决策前，评估项目风险和机会

## 使用步骤

### Step 1: 明确分析目标

**问题**: 我们要分析什么？分析的范围是什么？

**格式**:
```
分析对象：[公司/产品线/项目/个人]
分析范围：[时间范围、业务范围、地域范围]
分析目的：[战略规划/产品决策/竞争分析...]
```

**检查点**:
- [ ] 分析目标是否明确？
- [ ] 分析范围是否清晰？
- [ ] 是否有明确的分析目的？

### Step 2: 识别内部优势（Strengths）

**问题**: 我们有什么核心优势？我们比竞争对手做得好的是什么？

**维度**:
- **能力优势**：技术能力、研发实力、专业人才
- **资源优势**：资金、设备、供应链、渠道
- **品牌优势**：品牌知名度、用户口碑、合作伙伴关系
- **流程优势**：运营效率、成本控制、质量管理
- **文化优势**：团队凝聚力、创新文化、执行力

**格式**:
```
优势 (Strengths):
1. [优势1] - 具体表现：[数据/案例] - 重要性：[高/中/低]
2. [优势2] - 具体表现：[数据/案例] - 重要性：[高/中/低]
3. [优势3] - 具体表现：[数据/案例] - 重要性：[高/中/低]
...

核心优势（Top 3）：
1. ...
2. ...
3. ...
```

**检查点**:
- [ ] 优势是否有数据/案例支撑？
- [ ] 优势是否是真正的差异化能力？
- [ ] 优势是否可持续？

### Step 3: 识别内部劣势（Weaknesses）

**问题**: 我们有什么短板？我们在哪些方面落后于竞争对手？

**维度**:
- **能力短板**：技术短板、人才缺失、管理不足
- **资源短板**：资金短缺、设备老化、渠道不足
- **品牌短板**：品牌知名度低、用户口碑差
- **流程短板**：效率低下、成本过高、质量不稳定
- **文化短板**：团队涣散、创新不足、执行力差

**格式**:
```
劣势 (Weaknesses):
1. [劣势1] - 具体表现：[数据/案例] - 影响程度：[高/中/低]
2. [劣势2] - 具体表现：[数据/案例] - 影响程度：[高/中/低]
3. [劣势3] - 具体表现：[数据/案例] - 影响程度：[高/中/低]
...

核心劣势（Top 3）：
1. ...
2. ...
3. ...
```

**检查点**:
- [ ] 是否诚实面对劣势？
- [ ] 劣势是否可改善？
- [ ] 劣势对业务的影响程度？

### Step 4: 识别外部机会（Opportunities）

**问题**: 外部环境有哪些对我们有利的因素？市场趋势是什么？

**维度**:
- **市场机会**：市场增长、新细分市场、用户需求变化
- **技术机会**：新技术出现、技术成本下降
- **政策机会**：政策支持、法规放宽
- **竞争机会**：竞争对手失误、新进入者机会
- **合作机会**：潜在合作伙伴、生态机会

**格式**:
```
机会 (Opportunities):
1. [机会1] - 具体描述：[趋势/数据] - 可行性：[高/中/低]
2. [机会2] - 具体描述：[趋势/数据] - 可行性：[高/中/低]
3. [机会3] - 具体描述：[趋势/数据] - 可行性：[高/中/低]
...

高价值机会（Top 3）：
1. ...
2. ...
3. ...
```

**检查点**:
- [ ] 机会是否真实存在？
- [ ] 机会时间窗口有多长？
- [ ] 我们有能力抓住这个机会吗？

### Step 5: 识别外部威胁（Threats）

**问题**: 外部环境有哪些对我们不利的因素？可能面临什么风险？

**维度**:
- **竞争威胁**：新竞争对手、替代产品、价格战
- **市场威胁**：市场萎缩、用户需求变化
- **技术威胁**：技术颠覆、技术过时
- **政策威胁**：政策收紧、法规限制
- **运营威胁**：供应链风险、人才流失

**格式**:
```
威胁 (Threats):
1. [威胁1] - 具体描述：[趋势/数据] - 影响程度：[高/中/低]
2. [威胁2] - 具体描述：[趋势/数据] - 影响程度：[高/中/低]
3. [威胁3] - 具体描述：[趋势/数据] - 影响程度：[高/中/低]
...

高影响威胁（Top 3）：
1. ...
2. ...
3. ...
```

**检查点**:
- [ ] 威胁是否可能发生？
- [ ] 威胁的时间紧迫性？
- [ ] 我们有能力应对这个威胁吗？

### Step 6: 构建SWOT矩阵

**格式**:
```
                    优势 (S)                  劣势 (W)
                    [核心优势列表]            [核心劣势列表]
                    1. ...                    1. ...
                    2. ...                    2. ...
                    3. ...                    3. ...
───────────────────────────────────────────────────────────────
机会 (O)           │ SO策略（进攻型）         │ WO策略（补短板型）
[高价值机会列表]   │ 利用优势抓住机会         │ 克服劣势抓住机会
1. ...             │ 1. ...                   │ 1. ...
2. ...             │ 2. ...                   │ 2. ...
3. ...             │                          │
───────────────────────────────────────────────────────────────
威胁 (T)           │ ST策略（防御型）         │ WT策略（生存型）
[高影响威胁列表]   │ 利用优势应对威胁         │ 减少劣势避免威胁
1. ...             │ 1. ...                   │ 1. ...
2. ...             │ 2. ...                   │ 2. ...
3. ...             │                          │
```

### Step 7: 制定行动策略

**SO策略（进攻型）**:
```
策略1：[策略描述]
- 利用优势：[具体优势]
- 抓住机会：[具体机会]
- 行动计划：[具体步骤]
- 预期效果：[量化目标]
- 所需资源：[资源需求]
- 时间周期：[起止时间]
```

**WO策略（补短板型）**:
```
策略1：[策略描述]
- 克服劣势：[具体劣势]
- 抓住机会：[具体机会]
- 行动计划：[具体步骤]
- 预期效果：[量化目标]
- 所需资源：[资源需求]
- 时间周期：[起止时间]
```

**ST策略（防御型）**:
```
策略1：[策略描述]
- 利用优势：[具体优势]
- 应对威胁：[具体威胁]
- 行动计划：[具体步骤]
- 预期效果：[量化目标]
- 所需资源：[资源需求]
- 时间周期：[起止时间]
```

**WT策略（生存型）**:
```
策略1：[策略描述]
- 减少劣势：[具体劣势]
- 避免威胁：[具体威胁]
- 行动计划：[具体步骤]
- 预期效果：[量化目标]
- 所需资源：[资源需求]
- 时间周期：[起止时间]
```

### Step 8: 优先级排序

**评估维度**:
- **影响力**：对业务目标的影响程度（高/中/低）
- **可行性**：资源、能力、时间是否允许（高/中/低）
- **紧迫性**：时间窗口和风险程度（高/中/低）

**优先级矩阵**:
```
             高影响力
                 │
    优先执行     │   值得考虑
    (高可行性)   │   (低可行性)
─────────────────┼─────────────────
    暂缓         │   可有可无
    (低影响力)   │   (低影响力)
                 │
             低可行性
```

## 常见错误

- ❌ 优势和劣势混淆内部/外部因素
- ❌ 列出太多条目，没有优先级
- ❌ 没有数据支撑，主观臆断
- ❌ 只做分析，不制定策略
- ❌ 策略过于宽泛，缺乏可执行性
- ❌ 没有时间周期和责任人
- ❌ 只做一次，不定期更新
- ❌ 混淆现状和未来目标
