# Lean Canvas (精益画布)

## 目的

帮助创业公司快速梳理商业假设，识别最大风险，聚焦验证重点。相比商业模式画布，更关注问题、解决方案和竞争优势，更适合早期创业项目。

## 核心概念

```
┌──────────────┬──────────────┬──────────────────────┬──────────────┬──────────────┐
│   问题       │   解决方案   │      独特价值主张    │   不公平优势 │   客户细分   │
│  Problem     │  Solution    │  Unfair Advantage    │Unfair Adv.   │ Segments    │
│              │              │     UVP              │              │              │
│              │              │                      │              │              │
│  Top 3 问题  │  Top 3 功能  │  单句话价值主张      │  无法轻易复制 │  目标客户    │
│  现有替代方案 │              │  高概念              │  的竞争优势   │  早期采用者  │
│              │              │                      │              │              │
├──────────────┴──────────────┼──────────────────────┼──────────────┴──────────────┤
│        关键指标             │                      │         渠道通路              │
│    Key Metrics              │                      │         Channels             │
│                             │                      │                              │
│  关键业务指标               │                      │  触达客户的路径              │
│  (海盗指标)                 │                      │                              │
├─────────────────────────────┴──────────────────────┴──────────────────────────────┤
│                              成本结构 Cost Structure                                │
├───────────────────────────────────────────────────────────────────────────────────┤
│                              收入来源 Revenue Streams                               │
└───────────────────────────────────────────────────────────────────────────────────┘
```

**核心思想**:
- 问题 > 解决方案：先找到真正的问题
- 风险驱动：识别最大风险，优先验证
- 快速迭代：一页纸呈现，便于快速调整

## 何时使用

- ✅ 创业公司定义商业模式
- ✅ 新产品/新业务验证阶段
- ✅ 需要快速迭代商业模式假设
- ✅ 向投资者展示早期商业逻辑
- ✅ 识别最大风险并制定验证计划

## 使用步骤

### Step 1: 定义问题（Problem）

**问题**: 客户有什么痛点？现有解决方案有什么不足？

**格式**:
```
Top 3 问题：
1. [问题1] - 严重程度：[高/中/低] - 频率：[高/中/低]
2. [问题2] - 严重程度：[高/中/低] - 频率：[高/中/低]
3. [问题3] - 严重程度：[高/中/低] - 频率：[高/中/低]

现有替代方案：
1. [竞品1] - 解决了什么，还有什么不足
2. [竞品2] - 解决了什么，还有什么不足
3. [替代方案] - 用户当前的workaround
```

**检查点**:
- [ ] 问题是真实存在的吗？（验证过吗？）
- [ ] 问题够具体吗？（不是「用户体验不好」）
- [ ] 现有解决方案调研充分吗？

### Step 2: 定义客户细分（Customer Segments）

**问题**: 目标客户是谁？早期采用者是谁？

**分层**:
- 目标客户（Target Customer）：最终的大众市场用户
- 早期采用者（Early Adopters）：最先愿意尝试的用户

**格式**:
```
目标客户：
[人口特征]、[行为特征]、[心理特征]

早期采用者：
[具体描述他们是谁、在哪里、如何找到他们]
[他们的问题更严重、更迫切]

客户画像示例：
"张三，28岁，互联网公司产品经理，每天要处理大量需求文档，
现有的协作文档工具无法满足团队的特殊格式要求，不得不手动调整，
浪费大量时间。"
```

**检查点**:
- [ ] 早期采用者足够具体吗？
- [ ] 能通过什么渠道找到他们？
- [ ] 他们的问题足够痛吗？

### Step 3: 定义独特价值主张（Unique Value Proposition）

**问题**: 为什么客户应该选择你而不是竞争对手？你能提供什么独特价值？

**格式**:
```
单句话价值主张：
[目标客户] 使用 [产品名] 可以 [核心利益]，
相比 [替代方案]，我们 [差异化优势]。

高概念（High Concept）：
[知名产品A] + [知名产品B] = [你的产品]
或
"类似 [X]，但针对 [Y]"

示例：
"Slack for Healthcare" - 医疗行业的团队协作工具
"Uber for Dog Walking" - 宠物遛狗服务平台
```

**检查点**:
- [ ] 是否可以用一句话说清楚？
- [ ] 是否与问题直接对应？
- [ ] 是否有明确的差异化？

### Step 4: 定义解决方案（Solution）

**问题**: 如何解决Top 3问题？最简产品（MVP）应该包含什么功能？

**格式**:
```
Top 3 功能（针对Top 3问题）：
1. [功能1] → 解决问题1
   - 优先级：P0
   - MVP版本：[最简实现]

2. [功能2] → 解决问题2
   - 优先级：P0
   - MVP版本：[最简实现]

3. [功能3] → 解决问题3
   - 优先级：P1
   - MVP版本：[最简实现]
```

**检查点**:
- [ ] 每个功能是否直接对应问题？
- [ ] MVP范围是否足够小（可2周内完成）？
- [ ] 是否避免了功能堆砌？

### Step 5: 定义渠道通路（Channels）

**问题**: 如何触达早期采用者？如何触达大众用户？

**分层**:
- 早期渠道：触达早期采用者
- 规模化渠道：触达大众市场

**格式**:
```
早期渠道（免费/低成本）：
- [渠道1]：[具体做法] - 预期触达：[X人]
- [渠道2]：[具体做法] - 预期触达：[Y人]

规模化渠道（付费/规模化）：
- [渠道3]：[具体做法] - 预期CAC：[Z元]
- [渠道4]：[具体做法] - 预期CAC：[W元]
```

**检查点**:
- [ ] 早期渠道是否可执行？
- [ ] 是否有明确的获客成本预估？
- [ ] 渠道是否匹配目标客户？

### Step 6: 定义收入来源（Revenue Streams）

**问题**: 如何赚钱？定价策略是什么？

**格式**:
```
收入模式：[订阅/交易抽佣/广告/ licensing...]

定价策略：
- 免费版：[功能限制]
- 基础版：[价格] - [功能]
- 高级版：[价格] - [功能]
- 企业版：[价格] - [功能]

预估：
- 客单价：[X元]
- 付费转化率：[Y%]
- 月收入目标：[Z万元]
```

**检查点**:
- [ ] 收入模式是否可持续？
- [ ] 定价是否有市场依据？
- [ ] 是否有明确的付费转化目标？

### Step 7: 定义成本结构（Cost Structure）

**问题**: 启动需要多少钱？每月固定成本是多少？

**格式**:
```
一次性成本：
- [成本项1]：[金额]
- [成本项2]：[金额]

月固定成本：
- 人力：[人数] × [平均薪资] = [总额]
- 场地：[金额]
- 云服务：[金额]
- 其他：[金额]
- 小计：[总额]

月可变成本：
- 营销费用：[按用户计费]
- 云服务器：[按量计费]
- 其他：[金额]

烧钱率：[X万/月]
runway：[Y个月]
```

**检查点**:
- [ ] 是否遗漏隐性成本？
- [ ] runway是否足够（至少12-18个月）？
- [ ] 成本结构是否可扩展？

### Step 8: 定义关键指标（Key Metrics）

**问题**: 如何衡量产品是否成功？应该关注哪些关键指标？

**海盗指标（AARRR）**:
- 获取（Acquisition）：用户如何找到我们？
- 激活（Activation）：用户是否有良好的首次体验？
- 留存（Retention）：用户会回来吗？
- 收入（Revenue）：用户会付费吗？
- 推荐（Referral）：用户会推荐给别人吗？

**格式**:
```
关键指标：
- 获取：[新增用户数、渠道转化率...]
- 激活：[注册转化率、核心功能使用率...]
- 留存：[次日留存、7日留存、30日留存...]
- 收入：[付费转化率、ARPU、LTV...]
- 推荐：[NPS、邀请率、病毒系数...]

北极星指标：[最重要的单一指标]
```

**检查点**:
- [ ] 是否有明确的北极星指标？
- [ ] 指标是否可衡量、可追踪？
- [ ] 指标是否与业务目标对齐？

### Step 9: 定义不公平优势（Unfair Advantage）

**问题**: 什么是竞争对手无法轻易复制的？为什么你比他们更有优势？

**类型**:
- 技术壁垒（专利、算法）
- 网络效应（用户越多价值越大）
- 规模效应（成本优势）
- 品牌优势（信任度）
- 渠道优势（独家渠道）
- 数据优势（独家数据）
- 团队优势（领域专家）

**格式**:
```
不公平优势：
1. [优势1]：[具体描述] - 可持续性：[高/中/低]
2. [优势2]：[具体描述] - 可持续性：[高/中/低]

如果没有明显优势，说明：
"当前阶段没有明显不公平优势，计划通过 [方法] 建立"
```

**检查点**:
- [ ] 优势是否真的难以复制？
- [ ] 优势是否可持续？
- [ ] 如果没有优势，是否有建立优势的计划？

## 常见错误

- ❌ 问题不够具体，泛泛而谈
- ❌ 混淆目标客户和早期采用者
- ❌ 价值主张没有差异化
- ❌ 解决方案过于复杂，MVP范围过大
- ❌ 渠道不聚焦，试图触达所有人
- ❌ 没有不公平优势也不承认
- ❌ 关键指标过多，没有北极星指标
- ❌ 成本结构遗漏隐性成本
- ❌ 画布填完就放着，不验证假设

## 与其他方法的关系

- **前置方法**: user-interview（验证问题）、user-personas（定义客户）

- **并行方法**: business-model-canvas（成熟业务用）、startup-canvas（创业项目用）
