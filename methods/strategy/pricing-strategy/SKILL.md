# Pricing Strategy (定价策略)

## 目的

系统化设计产品定价策略，分析用户付费意愿、竞品定价、成本结构，制定合理的价格分层和定价策略，最大化收入和用户价值。

## 核心概念

```
定价策略框架：

┌─────────────────────────────────────────────┐
│           用户付费意愿                        │
│     用户愿意付多少钱？为什么？                │
├─────────────────────────────────────────────┤
│           定价模式                           │
│     采用什么定价模式？                        │
├─────────────────────────────────────────────┤
│           价格分层                           │
│     如何分层定价？                           │
├─────────────────────────────────────────────┤
│           定价策略                           │
│     如何动态调整价格？                        │
└─────────────────────────────────────────────┘

常见定价模式：
1. 免费增值（Freemium）
2. 订阅制（Subscription）
3. 按量计费（Usage-based）
4. 一次性付费（One-time）
5. 分层定价（Tiered）
6. 动态定价（Dynamic）
```

## 何时使用

- ✅ 新产品定价
- ✅ 重新定价或调价
- ✅ 设计价格分层
- ✅ 评估定价策略效果
- ✅ 向投资者展示盈利模式

## 使用步骤

### Step 1: 分析用户付费意愿

**问题**: 用户愿意付多少钱？为什么愿意付费？

**格式**:
```
用户付费意愿分析：

付费价值点：
1. [价值点1]
   - 价值描述：[用户获得什么价值]
   - 付费意愿：[高/中/低]
   - 价格敏感度：[高/中/低]
   - 价格接受区间：[X-Y元]
   - 用户占比：[XX%]

2. [价值点2]
   - ...

付费动机分析：
- 功能性动机：[为功能付费] - 占比[XX%]
- 情感性动机：[为情感付费] - 占比[XX%]
- 社会性动机：[为社会认同付费] - 占比[XX%]
- 便利性动机：[为便利付费] - 占比[XX%]

价格敏感度分析：
- 价格敏感型用户：[用户特征] - 占比[XX%]
- 价格中立型用户：[用户特征] - 占比[XX%]
- 价格不敏感型用户：[用户特征] - 占比[XX%]

支付能力分析：
- 目标用户收入水平：[收入范围]
- 可支配收入：[可支配收入]
- 类似产品支出：[用户在类似产品上的支出]

调研方法：
- [调研方法1]：[结果]
- [调研方法2]：[结果]
```

**检查点**:
- [ ] 是否识别了付费价值点？
- [ ] 是否有价格接受区间？
- [ ] 是否有用户调研支撑？

### Step 2: 分析竞品定价

**问题**: 竞品如何定价？市场定价水平如何？

**格式**:
```
竞品定价分析：

直接竞品：
1. [竞品1]
   - 定价模式：[订阅/一次性/按量...]
   - 价格分层：
     - 基础版：[价格] - [功能]
     - 高级版：[价格] - [功能]
   - 优势：[优势]
   - 劣势：[劣势]
   - 市场反馈：[用户评价]

2. [竞品2]
   - ...

间接竞品：
1. [替代品1]
   - 定价：[价格]
   - 对比：[与我们的差异]

市场定价水平：
- 最低价：[X元]
- 最高价：[Y元]
- 平均价：[Z元]
- 中位数：[W元]

定价趋势：
- 行业定价趋势：[趋势描述]
- 新兴定价模式：[模式描述]

差异化定价机会：
- [机会1]：[描述]
- [机会2]：[描述]
```

**检查点**:
- [ ] 是否分析了主要竞品？
- [ ] 是否有市场价格水平？
- [ ] 是否识别了差异化机会？

### Step 3: 分析成本结构

**问题**: 成本结构如何？定价需要覆盖什么成本？

**格式**:
```
成本结构分析：

固定成本：
- 人力成本：[X万/月]
- 场地成本：[Y万/月]
- 设备成本：[Z万/月]
- 其他固定成本：[W万/月]
- 小计：[V万/月]

可变成本：
- 云服务成本：[X元/用户/月]
- 客服成本：[Y元/用户/月]
- 支付成本：[Z%交易额]
- 其他可变成本：[W元/用户/月]

边际成本：
- 新增一个用户的边际成本：[X元]

单位经济：
- CAC（用户获取成本）：[X元]
- 服务成本：[Y元/用户]
- 最低定价（覆盖成本）：[Z元]

成本趋势：
- 随用户规模增长，成本如何变化：[描述]
```

**检查点**:
- [ ] 是否分析了固定和可变成本？
- [ ] 是否计算了边际成本？
- [ ] 是否计算了最低定价？

## 常见错误

- ❌ 定价缺乏依据，拍脑袋定价
- ❌ 忽视用户付费意愿调研
- ❌ 价格分层不合理，功能差异不明显
- ❌ 定价过高或过低
- ❌ 忽视竞品定价
- ❌ 忽视成本结构
- ❌ 定价策略过于复杂
- ❌ 只做规划，不验证假设
