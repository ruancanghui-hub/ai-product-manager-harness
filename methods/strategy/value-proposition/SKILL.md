# Value Proposition (价值主张)

## 目的

通过Jobs to be Done（JTBD）框架深入理解用户真正要完成的任务，设计与用户需求精准匹配的价值主张，确保产品真正解决用户问题并提供独特价值。

## 核心概念

```
Jobs to be Done (JTBD) 框架：

用户雇佣产品来完成某个任务：

┌────────────────────────────────────────────┐
│           用户要完成的任务 (Job)            │
│                                            │
│  "我需要[完成某个任务]                      │
│   因为[某种情境/动机]"                      │
└────────────────────────────────────────────┘
                     │
                     │ 雇佣
                     ▼
┌────────────────────────────────────────────┐
│              我们的产品                     │
│                                            │
│  功能性价值 + 情感性价值 + 社会性价值        │
└────────────────────────────────────────────┘

价值主张画布 (Value Proposition Canvas)：

┌──────────────────┬──────────────────────────┐
│   客户概况       │      价值主张            │
│ Customer Profile │  Value Proposition       │
├──────────────────┼──────────────────────────┤
│ 客户任务         │ 产品与服务               │
│ Customer Jobs    │ Products & Services      │
│                  │                          │
│ 痛点             │ 痛点缓解                 │
│ Pains            │ Pain Relievers           │
│                  │                          │
│ 收益期望         │ 收益创造                 │
│ Gains            │ Gain Creators            │
└──────────────────┴──────────────────────────┘
```

**三种价值**:
- **功能性价值**：解决具体问题
- **情感性价值**：满足情感需求
- **社会性价值**：满足社会认同需求

## 何时使用

- ✅ 新产品设计，明确价值主张
- ✅ 产品优化，重新审视价值
- ✅ 产品定位，找到差异化
- ✅ 营销传播，提炼核心卖点
- ✅ 用户调研后，总结用户需求
- ✅ 竞品分析后，找到差异化价值

## 使用步骤

### Step 1: 识别目标用户（Who）

**问题**: 我们为谁创造价值？

**格式**:
```
目标用户（Who）：

用户画像：
[用户姓名]，[年龄]岁，[职业]，[收入水平]

人口特征：
- 年龄：[年龄段]
- 性别：[性别]
- 地域：[地域]
- 职业：[职业]
- 收入：[收入水平]

行为特征：
- 使用场景：[在什么场景下使用]
- 使用频率：[多久使用一次]
- 消费习惯：[消费偏好]

心理特征：
- 价值观：[看重什么]
- 生活方式：[生活态度]
- 痛点强度：[高/中/低]

一句话描述：
"[用户]在[场景]下，想要[目标]，但[障碍]"
```

**检查点**:
- [ ] 用户画像是否足够具体？
- [ ] 是否有真实用户调研支撑？
- [ ] 是否区分了核心用户和边缘用户？

### Step 2: 理解用户任务（Why）

**问题**: 用户雇佣我们的产品来完成什么任务？

**JTBD六段式**:
1. **Who**: 谁
2. **Why**: 为什么（动机）
3. **What before**: 现状是什么
4. **How**: 如何解决
5. **What after**: 期望结果是什么
6. **Alternatives**: 现有替代方案

**格式**:
```
用户任务（Why）：

功能性任务（Functional Job）：
用户需要[完成什么具体任务]

示例：
- "我需要快速整理会议纪要"
- "我需要找到合适的供应商"
- "我需要监控服务器状态"

情感性任务（Emotional Job）：
用户想要[什么情感体验]

示例：
- "我想要感到掌控感"
- "我想要减少焦虑"
- "我想要获得认可"

社会性任务（Social Job）：
用户想要[什么社会认同]

示例：
- "我想要被同事认可为高效的人"
- "我想要在行业内建立专业形象"

任务陈述（JTBD Statement）：
"当[情境]时，我需要[完成任务]，
因为我想要[动机/期望结果]"

示例：
"当我开完一个重要会议时，我需要快速整理会议纪要，
因为我想要及时跟进待办事项，展现我的执行力"
```

**检查点**:
- [ ] 是否识别了功能性、情感性、社会性任务？
- [ ] 任务陈述是否具体？
- [ ] 是否有用户调研支撑？

### Step 3: 识别用户痛点（Pains）

**问题**: 用户在完成任务时遇到什么困难和阻碍？

**痛点类型**:
- **障碍**：阻止用户完成任务
- **不满**：现有方案的不足
- **风险**：用户担心的问题
- **成本**：时间、金钱、精力成本

**格式**:
```
用户痛点（Pains）：

痛点列表：
1. [痛点1]
   - 类型：[障碍/不满/风险/成本]
   - 具体描述：[场景+问题]
   - 严重程度：[高/中/低]
   - 发生频率：[高/中/低]
   - 用户原话："[用户访谈中的原话]"

2. [痛点2]
   - ...

痛点分类：

障碍（Obstacles）：
- [阻止用户完成任务的问题]

不满（Frustrations）：
- [现有方案的不足]

风险（Risks）：
- [用户担心的问题]

成本（Costs）：
- [时间、金钱、精力成本]

核心痛点（Top 3）：
1. [痛点1] - 严重程度：[高]
2. [痛点2] - 严重程度：[高]
3. [痛点3] - 严重程度：[中]
```

**检查点**:
- [ ] 是否识别了所有痛点类型？
- [ ] 是否有用户原话支撑？
- [ ] 是否排定了优先级？

### Step 4: 识别用户收益期望（Gains）

**问题**: 用户期望获得什么好处和结果？

**收益类型**:
- **必需收益**：必须满足的
- **期望收益**：希望满足的
- **渴望收益**：惊喜的
- **潜在收益**：未预期的

**格式**:
```
用户收益期望（Gains）：

收益列表：
1. [收益1]
   - 类型：[必需/期望/渴望/潜在]
   - 具体描述：[期望的结果]
   - 重要性：[高/中/低]
   - 用户原话："[用户访谈中的原话]"

2. [收益2]
   - ...

收益分类：

必需收益（Required Gains）：
- [必须满足的收益，不满足就失败]

期望收益（Expected Gains）：
- [用户期望的基本收益]

渴望收益（Desired Gains）：
- [超出期望的收益，会带来惊喜]

潜在收益（Potential Gains）：
- [用户未预期但会喜欢的收益]

核心收益期望（Top 3）：
1. [收益1] - 重要性：[高]
2. [收益2] - 重要性：[高]
3. [收益3] - 重要性：[中]
```

**检查点**:
- [ ] 是否识别了所有收益类型？
- [ ] 是否有用户原话支撑？
- [ ] 是否排定了优先级？

### Step 5: 定义产品与服务（Products & Services）

**问题**: 我们提供什么产品和服务来满足用户需求？

**格式**:
```
产品与服务：

核心产品：
- 产品名称：[产品名称]
- 产品描述：[功能描述]
- 解决的任务：[对应的JTBD]

支持服务：
1. [服务1]：[服务描述]
2. [服务2]：[服务描述]

产品特性：
1. [特性1]：[特性描述] → 对应用户任务：[任务]
2. [特性2]：[特性描述] → 对应用户任务：[任务]

产品如何帮助用户：
[描述产品如何帮助用户完成任务]
```

**检查点**:
- [ ] 产品是否直接对应用户任务？
- [ ] 是否区分了核心产品和支持服务？
- [ ] 是否避免了功能堆砌？

### Step 6: 定义痛点缓解（Pain Relievers）

**问题**: 我们的产品如何缓解用户的痛点？

**格式**:
```
痛点缓解：

痛点-缓解映射：
| 痛点 | 缓解方式 | 缓解程度 |
|-----|---------|---------|
| [痛点1] | [产品如何缓解] | [完全/部分/轻微] |
| [痛点2] | [产品如何缓解] | [完全/部分/轻微] |
| [痛点3] | [产品如何缓解] | [完全/部分/轻微] |

痛点缓解详情：
1. [痛点1]
   - 缓解方式：[具体描述]
   - 缓解程度：[完全/部分/轻微]
   - 产品特性：[哪个特性缓解了这个痛点]

2. [痛点2]
   - ...

核心痛点缓解（Top 3）：
1. [痛点1] → [缓解方式] → 缓解程度：[完全]
2. [痛点2] → [缓解方式] → 缓解程度：[部分]
3. [痛点3] → [缓解方式] → 缓解程度：[部分]
```

**检查点**:
- [ ] 是否缓解了核心痛点？
- [ ] 缓解方式是否具体？
- [ ] 是否有完全缓解的痛点？

### Step 7: 定义收益创造（Gain Creators）

**问题**: 我们的产品如何为用户创造收益？

**格式**:
```
收益创造：

收益-创造映射：
| 收益期望 | 创造方式 | 创造程度 |
|---------|---------|---------|
| [收益1] | [产品如何创造] | [完全/部分/轻微] |
| [收益2] | [产品如何创造] | [完全/部分/轻微] |
| [收益3] | [产品如何创造] | [完全/部分/轻微] |

收益创造详情：
1. [收益1]
   - 创造方式：[具体描述]
   - 创造程度：[完全/部分/轻微]
   - 产品特性：[哪个特性创造了这个收益]

2. [收益2]
   - ...

核心收益创造（Top 3）：
1. [收益1] → [创造方式] → 创造程度：[完全]
2. [收益2] → [创造方式] → 创造程度：[部分]
3. [收益3] → [创造方式] → 创造程度：[部分]
```

**检查点**:
- [ ] 是否创造了核心收益？
- [ ] 创造方式是否具体？
- [ ] 是否有惊喜收益？

### Step 8: 差异化价值主张

**问题**: 相比替代方案，我们的独特价值是什么？

**格式**:
```
差异化价值主张：

替代方案：
1. [替代方案1]
   - 方案描述：[描述]
   - 优势：[优势]
   - 劣势：[劣势]

2. [替代方案2]
   - ...

我们的差异化优势：
1. [差异化点1]
   - 具体描述：[描述]
   - 相比替代方案：[对比]
   - 用户感知度：[高/中/低]
   - 可复制性：[难/中/易]

2. [差异化点2]
   - ...

核心价值主张（一句话）：
针对[目标用户]，我们提供[产品/服务]，
帮助他们[完成什么任务]，
相比[替代方案]，我们更[差异化优势]。

价值主张陈述（完整版）：
[2-3段话阐述价值主张，包括用户、任务、痛点和收益]
```

**检查点**:
- [ ] 是否有明确的差异化？
- [ ] 差异化是否可持续？
- [ ] 是否可以用一句话说清？

## 常见错误

- ❌ 只关注功能性价值，忽视情感性和社会性价值
- ❌ 任务描述过于宽泛，不够具体
- ❌ 痛点和收益混淆
- ❌ 产品特性与用户任务不对应
- ❌ 价值主张缺乏差异化
- ❌ 主观臆断，缺乏用户调研支撑
- ❌ 价值主张过于复杂，无法一句话说清
- ❌ 只做分析，不验证假设
