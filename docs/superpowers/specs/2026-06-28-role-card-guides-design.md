# 角色卡详细使用指南设计规格

## 目标

为 H5 抽卡游戏的 43 张角色卡补充独立、可执行的岗位介绍，让用户不仅知道角色“会什么技能”，还知道“日常能帮我做什么”以及“怎样提供信息才能让他正确使用这些能力”。

## 内容模型

每张卡新增 `profile`，包含以下字段：

1. `summary`：1-2 句岗位定位，说明该角色负责解决哪类问题。
2. `dailyWork`：3 项日常可协助完成的具体工作，使用业务语言，不写抽象职责。
3. `howToUse`：3 步调用方法，明确用户需要提供的输入、应该提出的任务以及应要求的输出。
4. `promptExample`：一条可直接照着说的角色召唤口令，包含任务背景、目标和交付物。

43 张卡必须拥有独立内容。允许复用统一结构，不允许复制同一段正文或仅替换岗位名称。

## 页面呈现

### 抽卡揭示页

- 保留现有三项核心技能。
- 在技能下方新增“岗位详细介绍”。
- 直接列出岗位简介与“日常能帮你做什么”三项内容。
- 提供“查看完整使用指南”按钮，打开完整详情。
- 收藏和“再抽一张”继续保持可见、可操作。

### 卡牌详情弹窗

- 展示角色卡、三项技能和完整岗位使用指南。
- 使用指南按以下顺序展示：
  1. 岗位定位
  2. 日常能帮你做什么
  3. 如何让他使用这些能力
  4. 可直接使用的召唤口令
- 召唤口令使用独立强调区域，便于阅读和复制。

### 收藏与技能包

- 卡牌网格维持当前密度。
- 点击“查看技能”后进入同一个完整详情弹窗，无需新增页面或路由。

## 交互与响应式

- 桌面短屏：详情区域可以内部滚动，卡牌和主要操作不能被底部导航遮挡。
- 手机端：内容按技能、岗位介绍、调用方法纵向排列，页面自然滚动。
- 弹窗支持点击关闭按钮或遮罩关闭。
- 本次不增加后端、账号同步、内容编辑器或 AI 在线生成。

## 验收标准

1. 43 张卡都具有完整 `summary`、3 项 `dailyWork`、3 项 `howToUse` 和 `promptExample`。
2. UI 设计师卡明确包含高保真界面、视觉规范、组件/标注等实际工作内容。
3. 抽卡后不打开弹窗也能看到岗位简介和日常工作。
4. 完整详情能够读到调用步骤和召唤口令。
5. 收藏页与技能包打开的详情内容一致。
6. 桌面与手机尺寸无横向溢出，现有抽卡、收藏和筛选功能不回归。

## 测试策略

- 数据测试：检查 43 张卡字段完整、数组数量正确、召唤口令非空且不存在整段重复。
- 组件测试：抽卡后显示岗位介绍；打开详情后显示调用步骤与召唤口令。
- 浏览器验证：桌面与手机分别跑通抽卡、查看详细介绍和关闭弹窗。
