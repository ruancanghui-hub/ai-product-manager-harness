# 三、研发技术团队 — 日常工作总结

> 把设计图纸写成可运行的代码，APP 的骨骼和血肉

人数最多、分工最细的阶段。分为**客户端/前端开发**、**后端服务开发**、**基础技术支撑**三大板块。

---

## 一、客户端/前端开发（APP 本体核心）

### 1. iOS 开发工程师

**一句话定位**：用 Swift/OC 把 iPhone/iPad APP 写出来，搞定苹果生态一切。

**日常：**
- **功能开发**：拿到需求+设计稿→技术方案设计→编码实现→自测→提测；写 ViewController（页面）、Model（数据）、Network（网络请求）、Cache（缓存）等各层代码
- **UI 还原**：对照设计稿精确还原（像素对齐、颜色一致、间距一致）；实现复杂手势交互和列表流畅滚动
- **系统适配**：适配新 iOS 版本 API 变更（每年 WWDC 后适配）；适配 iPhone/iPad 不同尺寸、横竖屏、分屏、灵动岛；适配深色模式 Dynamic Color、SF Symbols
- **App Store 上架**：准备元数据（截图/描述/关键词）、处理苹果审核反馈；接入苹果支付 IAP（消耗型/非消耗型/订阅型）、Sign in with Apple
- **性能优化**：排查内存泄漏（Instruments 检测 retain cycle）；优化启动速度（二进制重排、减少动态库加载）；列表/页面帧率优化（离屏渲染检测）
- **日常协作**：参加每日站会，同步进度；提测前和测试沟通测试点；Bug 修复（测试提的 + 线上 Crash 监控）

**技术栈：** Swift / SwiftUI（趋势） / Objective-C（存量代码）/ UIKit / Swift Concurrency（async/await）/ Combine / Alamofire / Kingfisher

**大厂差异：**
- **苹果（Apple 内部）**：iOS 开发用内部工具链（非公开），代码审查极其严格，性能要求极高
- **阿里/腾讯**：大团队 iOS 开发 20-50+ 人，分业务模块（首页/详情/购物车/支付各一人），有专门的组件化基建

---

### 2. Android 开发工程师

**一句话定位**：用 Kotlin/Java 写安卓 APP，适配国内厂商+Google Play。

**日常：**
- **功能开发**：用 MVVM/MVI 架构写页面，ViewModel + LiveData/Flow + Room（数据库）+ Retrofit（网络）；对照设计稿实现 UI，处理 Android 百种屏幕分辨率适配
- **碎片化适配**：适配不同 Android 版本（API Level 差异，minSdk→targetSdk）；适配不同厂商 ROM（小米/华为/OPPO/vivo 的权限管理、通知渠道、后台限制各不相同）；适配不同屏幕类型（挖孔屏/刘海屏/瀑布屏/折叠屏）
- **性能优化**：启动优化（MultiDex、懒加载）；内存优化（LeakCanary 检测泄漏、大图处理）；包体积瘦身（资源压缩、混淆、移除无用库）；ANR 和卡顿排查（Systrace/Profiler）
- **Google Play 上架**：AAB 格式打包、Play 政策合规、目标 API Level 要求、隐私清单
- **国内渠道**：应用宝/华为/小米/OPPO/vivo 应用商店上架，各平台审核规则不同
- **日常协作**：和 iOS 同步开发进度、接口联调、Bug 修复

**技术栈：** Kotlin / Java / Jetpack Compose（趋势）/ ViewModel + LiveData + Flow / Retrofit + OkHttp / Room / Glide / Coroutines

**大厂差异：**
- **谷歌（Android 团队）**：Android 源码贡献者众多，内部开发标准极高
- **国内大厂**：Android 开发人数通常多于 iOS（安卓市场占有率高），需要专人维护厂商适配

---

### 3. 跨端开发工程师

**一句话定位**：写一套代码，同时跑 iOS + Android，降本增效。

**日常：**
- **功能开发**：用 Flutter（Dart）或 React Native（TS/JS）开发页面，同时产出双端 APP；接入双端原生能力（相机、定位、推送、支付）通过 Bridge/Channel 调用
- **性能调优**：列表长列表优化（懒加载、回收复用）；首屏加载速度优化（预加载、骨架屏）；动画流畅度（60fps，避免 JS-Native Bridge 通信过多）
- **UI 还原**：跨端 UI 组件和原生体验一致性问题处理（iOS 和 Android 风格差异）；自定义复杂 UI（图表、手势）时原生能力的调用
- **工程化**：维护跨端项目工程结构、组件化拆分；管理依赖和版本升级（Flutter SDK、RN 版本）；CI/CD 流水线集成的双端打包
- **日常协作**：和原生团队协作（原生模块暴露给跨端用）；和测试协作适配双端测试

**技术栈：** Flutter (Dart) / React Native (TypeScript) / uni-app (Vue)

**大厂差异：**
- **阿里**：闲鱼/淘特大量用 Flutter，有自研 Flutter 引擎优化；Weex（曾经自研跨端方案）
- **腾讯**：微信小程序是最成功的「跨端」方案之一；部分业务用 Flutter/RN
- **谷歌**：Flutter 是谷歌出品，内部大量使用并推动生态
- **注意**：核心 APP（微信/淘宝）主链路还是原生为主，跨端用于次级页面或独立小 APP

---

### 4. 鸿蒙开发工程师

**一句话定位**：华为 HarmonyOS APP 开发，国产替代新增岗位。

**日常：**
- **功能开发**：用 ArkTS + ArkUI 写鸿蒙 APP 页面；实现鸿蒙特有能力：原子化服务（免安装卡片）、服务卡片、多设备流转（手机→平板→车机无缝切换）
- **系统适配**：适配鸿蒙不同版本 API；适配鸿蒙的权限模型（和安卓不同）；处理鸿蒙的系统通知、后台任务机制
- **跨设备流转**：实现「在一台设备打开，另一台设备继续用」的跨端连续体验；分布式数据管理（跨设备数据同步）
- **应用市场上架**：华为应用市场（AppGallery）上架审核、鸿蒙应用资质
- **日常协作**：和 Android/iOS 团队对齐功能同步策略

**技术栈：** ArkTS / ArkUI / DevEco Studio / HarmonyOS SDK

**大厂差异：**
- **华为自身**：鸿蒙开发团队最大，核心系统+生态建设
- **阿里/腾讯**：2024 年后加速鸿蒙适配，主要 APP（淘宝/微信）必须适配鸿蒙
- **外企**：暂不涉及

## 二、后端服务开发

### 5. 后端开发工程师（Java/Go/C++）

**一句话定位**：写所有数据的接口——登录、订单、消息、内容……APP 连的就是后端。

**日常：**
- **接口开发**：设计 RESTful API / gRPC 接口（路径/参数/返回值/错误码）；写业务逻辑代码（用户模块、订单模块、支付模块、内容模块、消息模块）；接入中间件（缓存 Redis、消息队列 Kafka/RocketMQ、搜索 ES）
- **数据库设计**：设计库表结构，写 SQL，优化慢查询；处理事务和并发（分布式锁、乐观锁/悲观锁）
- **接口文档**：写接口文档（Swagger/YAPI/内部平台），前端开发照此对接
- **性能优化**：接口响应时间优化（加缓存、异步处理、并行调用）；高并发处理（限流、降级、熔断）；慢 SQL 优化（索引、分库分表）
- **线上维护**：线上报错日志排查（ELK/Sentry）；告警响应（P0/P1/P2 分级处理）；数据修复（异常数据批量修复脚本）
- **日常协作**：和客户端对齐接口字段；和大数据团队对接日志数据；和 DBA 评审库表变更

**技术栈：** Java (Spring Boot) / Go / C++ / Python / Node.js / Redis / MySQL / Kafka / Elasticsearch / Docker

**大厂差异：**
- **阿里**：Java 系为主，HSF（自研 RPC）+ 内部中间件体系完备
- **腾讯**：C++/Go 系比重大，自研 TAF（RPC 框架）
- **谷歌**：C++/Go/Java 内部混合，内部基建（Borg/Spanner/Protocol Buffers）统一
- **中小公司**：Java Spring Boot + MySQL + Redis 一套走天下

---

### 6. 数据库工程师 DBA

**一句话定位**：管好公司的数据资产——不丢、不慢、不乱。

**日常：**
- **日常运维**：监控数据库性能（CPU/IO/连接数/慢查询）；处理数据库故障（死锁、主从延迟、磁盘满）；备份恢复演练
- **SQL 优化**：审查开发提交的 SQL 语句，指出全表扫描/索引失效等问题；分析慢查询日志，优化 TOP 慢 SQL；指导开发合理设计索引
- **架构管理**：分库分表策略（水平拆分/垂直拆分）；读写分离、主从复制、多活架构；数据归档和冷热分离
- **变更管理**：审核开发的上线 SQL，在测试环境验证，生产环境执行；大表 DDL 变更（在线改表工具 pt-osc/gh-ost）
- **日常协作**：和后端开发评审库表设计；和运维对接服务器资源；和架构师规划数据库扩展方案

**技术栈：** MySQL / Redis / MongoDB / TiDB / PostgreSQL / 阿里云 RDS / AWS RDS / 腾讯云 CDB

**大厂差异：**
- **阿里**：OceanBase（自研分布式数据库）用于金融/支付场景；DBA 团队独立
- **谷歌**：Spanner（自研全球分布式数据库）+ F1，不依赖开源方案
- **中小公司**：后端兼做 DBA，到一定规模才招专职

---

### 7. 大数据开发工程师

**一句话定位**：把 APP 每天产生的海量用户行为数据存起来、算出来、用起来。

**日常：**
- **数据采集**：搭建埋点日志采集链路（APP→日志服务→数据仓库）；维护数据管道（日志清洗、格式转换、去重、异常过滤）
- **数据仓库**：设计数仓分层（ODS→DWD→DWS→ADS）；写 ETL 任务——每天凌晨把数据从原始层加工到汇总层；维护数据字典和数据血缘
- **实时计算**：搭建实时数据流（Flink/Spark Streaming/Kafka Streams）；实时大屏数据（实时 UV、订单、收入）；实时风控——异常行为秒级拦截
- **计算平台**：管理 Hadoop/Spark/Flink 集群资源；优化计算任务性能（数据倾斜、内存溢出）
- **日常协作**：给数据分析师提供干净可用的数据表；给算法工程师提供训练数据；和 PM/用研配合做用户行为分析

**技术栈：** Hadoop / Spark / Flink / Kafka / Hive / HBase / ClickHouse / 阿里云 MaxCompute / AWS EMR

**大厂差异：**
- **阿里**：MaxCompute + DataWorks 自有生态
- **腾讯**：TDW（自研数仓平台）
- **谷歌**：BigQuery + Dataflow 云原生，内部不做集群运维
- **中小公司**：无此岗位，埋点数据靠第三方（神策等）

## 三、基础技术支撑岗

### 8. 客户端架构师

**一句话定位**：APP 的技术地基——组件化、性能兜底、工程效率。

**日常：**
- **架构设计**：设计 APP 整体架构（模块拆分、组件化方案、路由设计）；制定技术规范（代码风格、命名规范、目录结构、分支管理）；技术选型评审（新库/新框架引入）
- **工程效率**：搭建组件化工程结构（各模块独立编译/调试）；优化编译速度（增量编译、远程编译）；维护 CI 打包流水线
- **性能治理**：启动速度专项优化、包体积瘦身、内存治理、卡顿/ANR 治理；搭建性能监控平台（启动/页面加载/帧率上报）
- **日常协作**：评审各业务开发的技术方案；解决开发提出的疑难技术问题；新人技术培训和技术分享

**技术栈（iOS）：** Xcode / CocoaPods / SPM / iOS 底层（Runtime/Runloop/内存管理）
**技术栈（Android）：** Gradle / Transform/ASM / Android 底层（Binder/Handler/AMS）

---

### 9. 服务端架构师

**一句话定位**：后端系统的顶层设计师——扛住百万并发、保证数据一致。

**日常：**
- **架构设计**：设计分布式系统架构（微服务拆分、服务治理）；设计技术方案：缓存策略、消息可靠性、分布式事务、最终一致性；画系统架构图、时序图，写技术方案评审文档
- **容量规划**：预估业务增长量 → 服务器扩容计划；全链路压测 → 找瓶颈 → 优化；多活/容灾架构设计
- **技术评审**：评审各业务线的重大技术方案（新功能/新系统）；对高风险方案提出改进建议
- **日常协作**：和服务端开发讨论技术难点（分布式锁用 Redis 还是 ZooKeeper）；和 DBA 规划存储层扩容；和技术负责人对齐架构演进方向

---

### 10. 音视频工程师

**一句话定位**：直播/短视频/语音通信的核心技术——低延迟、高清晰。

**日常：**
- **直播/推流**：优化推流端（编码参数、码率自适应、弱网对抗）；播放器开发优化（首帧时间、卡顿率）；低延迟方案（WebRTC/RTMP/QUIC）
- **音视频处理**：美颜/滤镜 SDK 集成和效果调优；音频 3A（AEC 回声消除/ANS 降噪/AGC 自动增益）；视频编辑（裁剪/变速/转场/贴纸）
- **编解码**：H.264/H.265/H.266 编码优化（软编/硬编选择）；音频编解码（AAC/Opus）
- **传输优化**：CDN 调度、弱网传输协议（QUIC/SRT）；实时通信延迟优化（<200ms）
- **日常协作**：和客户端开发对接 SDK 接口；和算法团队对接美颜/超分模型

**技术栈：** FFmpeg / WebRTC / MediaCodec / VideoToolbox / OpenGL / Metal

**大厂差异：**
- **抖音/快手/腾讯视频**：音视频团队规模大（几十到上百人），自研编解码器和传输协议
- **YouTube（谷歌）**：VP9/AV1 自研编解码，传输协议极度优化
- **中小公司**：直接用第三方 SDK（七牛/腾讯云/阿里云音视频），不做底层

---

### 11. 安全工程师

**一句话定位**：防止 APP 被攻击、数据被泄露、账号被盗。

**日常：**
- **APP 安全**：防逆向（代码混淆/加固/反调试）；防抓包（HTTPS Pinning/证书校验）；防重打包（签名校验/完整性校验）；敏感数据保护（Keychain/Keystore 加密存储）
- **接口安全**：接口防刷（限流/验证码/人机验证）；敏感接口加密传输和签名校验；防止越权访问（水平越权/垂直越权检测）
- **业务安全**：防刷单/薅羊毛（设备指纹/行为特征分析）；账号安全（撞库检测/异地登录告警/二次验证）；支付安全（支付流程安全审计）
- **隐私合规**：个人信息采集最小化审计；隐私政策合规检查；数据跨境传输合规
- **安全测试**：渗透测试（模拟攻击找漏洞）；代码安全扫描（SAST/DAST）；第三方 SDK 安全审查
- **应急响应**：线上安全事件应急处理和溯源

**技术栈：** 逆向工具（IDA Pro/Frida/Xposed）/ 加解密 / 渗透测试工具 / 代码扫描

**大厂差异：**
- **阿里/腾讯**：安全团队是独立部门（阿里安全/腾讯安全应急响应中心 TSRC），几十上百人
- **谷歌**：Project Zero 是顶级安全团队，Android 安全有专门的平台安全团队
- **苹果**：iOS 安全是系统级封闭，安全团队庞大但对内外部高度封闭
- **中小公司**：无专职安全，依赖第三方安全服务

---

### 12. 算法工程师

**一句话定位**：给 APP 加「智能」——推荐、识别、理解、生成。

**日常：**
- **推荐算法**：用户画像构建（兴趣标签、行为序列）；召回→粗排→精排→重排的推荐链路优化；模型训练与迭代（DeepFM/Wide&Deep/DIN/Transformer）；AB 实验评估推荐效果（点击率/时长/转化）
- **CV（计算机视觉）**：图像/视频内容理解（审核/标签/相似去重）；人脸识别/美颜/AR 特效；OCR 文字识别
- **NLP/语音**：ASR 语音识别 → 转文字；NLU 意图识别 → 智能客服/搜索；文本审核（垃圾评论/违规内容检测）
- **大模型/AIGC**：接入/微调大模型用于搜索、推荐、客服、内容生成
- **日常协作**：和推荐工程团队（偏工程）对接模型上线；和数据团队对接训练数据；和产品/分析对接实验指标

**技术栈：** Python / TensorFlow / PyTorch / 特征平台 / 模型服务平台

**大厂差异：**
- **阿里**：阿里妈妈（广告算法）、淘宝搜索推荐算法团队，达摩院做前沿研究
- **腾讯**：微信/QQ 的搜一搜推荐、游戏 AI，AI Lab / 优图实验室
- **谷歌**：Research + Product 双层，DeepMind/Google Research 学术前沿，产品算法团队做落地
- **苹果**：设备端 ML 能力（CoreML/Neural Engine），重隐私保护的端侧推理
- **中小公司**：用第三方 AI 服务（阿里云/腾讯云 AI API），无自研

---

### 13. DevOps / 运维工程师

**一句话定位**：保证服务器不挂、发布不翻车、出问题快速恢复。

**日常：**
- **CI/CD**：维护代码→构建→测试→打包→发布的自动化流水线（Jenkins/GitLab CI/GitHub Actions）；配置多环境（开发/测试/预发/生产）
- **容器/编排**：Docker 镜像管理；Kubernetes 集群管理（扩容/缩容/滚动更新/服务发现）
- **线上运维**：监控告警体系（Prometheus + Grafana / 云监控）；日志收集分析（ELK / Loki）；7×24 值班，收到告警→定位→止损→复盘
- **发布管理**：灰度发布（先 1%→5%→100%）；发布检查（核心指标是否异常）；发布回滚预案

**技术栈：** Linux / Docker / Kubernetes / Jenkins / Terraform / Prometheus / Grafana / ELK

**大厂差异：**
- **阿里/腾讯**：运维自动化程度高，有自研运维平台（阿里 StarAgent/腾讯蓝鲸）
- **谷歌**：SRE（Site Reliability Engineer）概念发源地，软件工程能力解决运维问题，50% 时间写代码
- **苹果**：基础设施极其庞大但外界了解很少，极度重视服务可靠性和隐私保护
- **中小公司**：后端兼运维，或全部上云（阿里云/腾讯云/AWS）减少运维负担

### 14. 编译/打包工程师

**一句话定位**：让 APP 从代码变成能装的包——快、稳、自动化。

**日常：**
- **构建优化**：优化编译速度（增量编译、分布式编译、远程构建缓存）；模块化构建（按模块独立编译，改一行只编译一个模块）
- **打包流水线**：维护多渠道打包（国内几十个渠道 + Google Play）；自动签名管理（证书/描述文件自动匹配）；Diamond 构建（一次构建产出多包）
- **版本管理**：维护版本号规则（major.minor.patch.build）；管理 Git Tag 和发布分支；管理依赖版本（锁版本 vs 自动更新）
- **日常协作**：和 iOS/Android 开发对接编译问题；和 DevOps 对接 CI/CD 集成

**技术栈：** Gradle / Bazel / Xcode Build / Fastlane / CocoaPods / SPM

**大厂差异：**
- **谷歌**：Bazel（自研构建系统）统一全公司构建，速度极快
- **阿里/腾讯**：有专门的构建平台团队，百人规模 APP 编译一次从 1 小时优化到 10 分钟内
- **中小公司**：Fastlane 一键搞定

### 15. 测试开发工程师

**一句话定位**：写工具让测试更自动化——不是测功能，是造轮子让测试更高效。

**日常：**
- **自动化框架**：搭建 UI 自动化测试框架（iOS: XCTest/XCUITest, Android: Espresso/UiAutomator）；搭建接口自动化测试框架（pytest + requests / JUnit + RestAssured）
- **测试工具**：开发造数据工具；开发 Mock 服务（假接口）；开发弱网模拟/机型适配自动化工具
- **CI 集成**：把自动化测试集成到 CI 流水线，每次提交/每天凌晨自动跑；自动化测试报告生成（通过率、失败详情、截图）
- **性能测试平台**：开发启动性能测试工具、内存监控工具；开发压测平台（QPS 阶梯加压）
- **日常协作**：和功能测试沟通哪些用例可以自动化；和 DevOps 对接 CI 集成

**技术栈：** XCTest / Espresso / Appium / pytest / Jenkins

**大厂差异：**
- **阿里/腾讯**：测试开发团队和功能测试团队分开，测开偏工具开发
- **谷歌**：测试开发能力极强，大部分测试自动化，手工测试很少
- **中小公司**：无此岗位，功能测试兼做
