# 二、设计团队 — 日常工作总结

> 把需求变成看得见、点得到的界面

---

## 1. 交互设计师 UX

**一句话定位**：设计「用户怎么用」，保证操作顺畅不迷路。

### 📋 日常核心工作

| 时间段 | 工作内容 |
|--------|----------|
| **需求评审阶段** | 参加 PM 的需求评审会，理解业务目标和用户场景；对 PM 的逻辑提出交互层面质疑（"这个流程用户要7步，太长了"）；拿到 PRD 后拆解页面结构和跳转逻辑 |
| **设计阶段** | 画信息架构图（页面层级、模块关系）；画任务流程图（用户完成一个任务的完整路径）；画页面交互稿（线框图/高保真原型），标注：每个按钮点击后发生什么、每个输入框的校验规则、空页面/加载中/错误/网络异常的展示；设计手势交互（滑动删除、下拉刷新、长按菜单）、动效逻辑描述（给动效设计师的输入） |
| **评审阶段** | 交互稿评审：邀请 PM + 视觉 UI + 开发一起过，对齐交互逻辑可行性；根据反馈修改，同步更新交互文档 |
| **开发跟进** | 开发过程中解答开发对交互细节的疑问；开发完成后走查（和交互稿对比，提 bug 给开发）；参与 UI 还原走查 |
| **日常积累** | 体验竞品 APP 交互模式，建交互组件库；关注 iOS Human Interface Guidelines / Android Material Design 更新 |

### 🔧 常用工具
- 交互原型：Figma / Sketch / Axure / Principle / ProtoPie
- 流程图：FigJam / Miro / Whimsical / draw.io
- 标注：蓝湖 / 摹客 / Zeplin

### 🏢 大厂差异
- **阿里/腾讯**：UX 属于 UED 部门，一个交互设计师跟 1-2 条业务线，有交互规范库沉淀
- **谷歌**：UX Designer 身兼交互+视觉，角色边界不僵化；Material Design 团队统一输出设计规范
- **苹果**：人机交互团队（HI Team）级别极高，重交互直觉和物理隐喻
- **中小公司**：UX 和 UI 是一个人，甚至 PM 画线框图就行

## 2. 视觉设计师 UI

**一句话定位**：决定 APP「长什么样」，好看且符合品牌。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **视觉设计** | 拿到 UX 交互稿后，做视觉填充：配色方案（主色/辅色/功能色/中性色）、字体层级（标题、正文、辅助文字）、图标风格、间距体系、圆角/阴影规范；设计所有页面视觉稿（含暗黑模式）；设计弹窗、Toast、Loading、空页面等组件 |
| **设计系统维护** | 在 Figma/Sketch 中维护组件库（Button、Input、Card、Tab、Modal 等）；更新设计规范文档，确保新老设计保持一致；对接开发，确保代码组件库和设计组件库 1:1 对应 |
| **切图标注** | 输出各分辨率的切图（@1x/@2x/@3x）；标注间距、颜色、字号、圆角、阴影，上传协作平台 |
| **视觉走查** | 开发实现后，逐个页面截图对比设计稿（像素级），标记差异点；提「UI Bug」给开发修复，不通过不准合代码 |
| **品牌物料** | APP Store 截图、应用介绍图、功能引导页、新功能宣传图 |
| **日常积累** | 收集设计灵感（Dribbble / Behance / Pinterest / Mobbin）；关注设计趋势（玻璃态、新拟物、极简风）；沉淀设计方法论 |

### 🔧 常用工具
- 设计：Figma / Sketch / Adobe XD / Photoshop / Illustrator
- 协作：蓝湖 / 摹客 / Zeplin / Figma 内建
- 素材：Dribbble / Behance / Pinterest / Mobbin / IconFont

### 🏢 大厂差异
- **阿里**：UI 设计师分平台（淘宝/天猫/支付宝各有独立设计团队），Ant Design 是 B 端设计标杆
- **腾讯**：TDesign 体系，ISUX 团队（社交用户体验设计部）负责 QQ/微信等核心产品
- **谷歌**：Material Design 是统一设计语言，所有 APP 在此框架内设计，视觉设计师参与组件库共建
- **苹果**：Human Interface Guidelines，设计团队庞大但对外的设计资产极其内敛
- **中小公司**：UI 一个人搞定所有视觉+切图，没有设计系统

## 3. 动效设计师

**一句话定位**：让 APP「活起来」，动效传达状态变化和品牌个性。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **页面动效** | 页面转场动画（Push/Pop/Present 的方向、速度、缓动曲线）；Tab 切换动效；列表加载动画（骨架屏→内容）；下拉刷新动画（品牌 IP 融入） |
| **微交互** | 按钮点击反馈（缩放/颜色变化/波纹扩散）；点赞/收藏/关注的情感化动效（撒花、弹跳）；开关、滑块、进度条的状态过渡 |
| **品牌动效** | APP 启动动画；Logo 演绎动画；活动页面入场动画；节日皮肤的动效设计 |
| **交付** | 在 AE/Principle 中制作动效 Demo → 标注参数（持续时间、缓动曲线、位移/透明度/缩放/旋转数值）→ 给开发实现；复杂动效用 Lottie 导出 JSON（开发可直接用） |
| **日常协作** | 和 UX 确认动效触发场景；和 UI 确认动效中的视觉元素；和开发沟通可实现性 |

### 🔧 常用工具
- 设计：After Effects / Principle / ProtoPie / Rive
- 交付：Lottie (Bodymovin) / 动效参数标注

### 🏢 大厂差异
- **阿里/腾讯**：大厂有专职动效设计师（UI 动效分开），尤其在直播/短视频/游戏 APP 中动效权重极高
- **谷歌/苹果**：动效融入设计系统（Material 的 motion guidelines），系统级动效有统一规范
- **中小公司**：UI 兼做简单动效，或用 Lottie 模板

## 4. 品牌/插画设计师

**一句话定位**：打造 APP 视觉辨识度，从图标到运营画面。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **APP 品牌** | 设计 APP 图标（含各尺寸适配）；启动页设计；应用商店图标和截图 |
| **插画体系** | 建立 APP 内插画风格（扁平/手绘/3D/C4D）；设计空状态插画（无数据、无网络、无权限）；引导页插画；新手引导页 |
| **运营视觉** | 活动 Banner、H5 活动页、节日/大促主题皮肤；开屏广告视觉；推送消息配图 |
| **营销物料** | 社交媒体宣传图、公众号封面、线下海报、EDM 邮件配图 |
| **日常协作** | 接收运营团队活动需求 → 输出设计方案 → 交付各尺寸物料；和 UI 设计师对齐品牌视觉规范 |

### 🔧 常用工具
- 插画：Procreate / Illustrator / Photoshop
- 3D：Cinema 4D / Blender
- 排版：InDesign / Canva

### 🏢 大厂差异
- **阿里**：双11 级别的运营视觉团队极庞大，品牌插画师分场景（大促/频道/营销）
- **谷歌**：品牌插画相对标准化（Illustration System），各产品套统一的插画风格
- **苹果**：极少插画装饰，以产品摄影和极简文字为主
- **中小公司**：UI 兼做或外包

---

## 5. 多端适配设计师

**一句话定位**：让 APP 在手机、平板、折叠屏、车机上都能用。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **多尺寸适配** | 制定适配规则：不同屏幕宽度下的布局方案（单列/双列/三列）；断点设计（小屏手机/大屏手机/平板/折叠屏展开）；弹性布局和自适应规则 |
| **平板适配** | iPad/Android 平板横竖屏布局设计；分屏/多窗口模式适配 |
| **折叠屏适配** | 折叠态→展开态的 UI 过渡方案；大屏多任务/平行视界布局 |
| **车机/Watch** | Apple Watch / Wear OS 表盘 APP 设计；CarPlay / Android Auto 车载界面设计 |
| **设计规范** | 输出多端适配设计规范文档，标注各端差异；和开发对齐适配规则（Autolayout/ConstraintLayout） |

### 🔧 常用工具
- Figma（Constraints/Auto Layout）/ Sketch（Resizing）

### 🏢 大厂差异
- **谷歌/苹果**：非常重视多设备生态（iPhone+iPad+Mac+Watch+TV / Android+Wear+TV+Auto），适配团队独立
- **国内大厂**：折叠屏适配是近2年新增需求，华为/OPPO/小米的折叠屏都要适配
- **中小公司**：只做手机端，UI 兼做简单适配
