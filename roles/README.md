## 🔄 角色协作全景图

```
                        ┌── 总调度 ──┐
                        │  盯节奏/依赖/瓶颈/信息/质量  │
                        └─────┬─────┘
                              │
产品中心（定方向）              │
    ↓ PRD                     │
设计团队（出图纸）  ←──────────┤ 环环咬合
    ↓ 视觉稿/交互稿             │
研发团队（写代码）← 架构师评审   │ 不脱节
    ↓ 提测包                   │
测试团队（找bug）              │
    ↓ 测试通过                 │
DevOps（打包上线）             │
    ↓ 灰度→全量                │
运营/商业化（拉新赚钱）         │
    + 客服（用户反馈）          │
    ↓ 数据回收                 │
数据分析师 → PM → 下一迭代 ────┘
```

## 🏢 大厂 vs 中小团队差异速览

| 维度 | 中小团队（5-10人） | 大厂（50-200+人） |
|------|---------------------|---------------------|
| PM | 1人全包 | 按方向拆分（后台/商业化/海外） |
| 设计 | 1个UI兼UX | UX/UI/动效/品牌/多端 各专人 |
| 客户端 | 1-2人跨端 | iOS/Android/鸿蒙/跨端 各团队 |
| 后端 | 1-2人全栈 | Java/Go/C++ 分组 + DBA + 大数据 |
| 测试 | 开发自测为主 | 功能/自动化/性能/兼容/安全/合规 专职 |
| 支撑岗 | 无 | 架构师/音视频/安全/算法/DevOps 标配 |
| 运营 | PM兼做 | 用户/内容/渠道/商业化 独立团队 |
