# 四、测试团队 — 日常工作总结

> 保证 APP 不出 bug，用户体验不翻车

---

## 1. 功能测试工程师（手工测试）

**一句话定位**：APP 上线前的最后把关人，把所有功能亲手测一遍。

### 📋 日常核心工作

| 时间段 | 工作内容 |
|--------|----------|
| **需求评审阶段** | 参加需求评审会，理解本期需求；提前识别测试风险点（边界条件、异常场景） |
| **用例编写** | 拿到 PRD+设计稿后写测试用例：正常流程 + 异常流程 + 边界值 + 兼容场景；用例评审会——PD+开发+测试一起过，确保测试覆盖无遗漏 |
| **测试执行** | 搭建测试环境（测试服务器/测试账号/测试数据）；按用例逐条执行，标记通过/失败；发现 bug → 提交到 Jira/TAPD：标题、复现步骤、预期/实际结果、截图/录屏、日志、优先级（P0/P1/P2） |
| **特殊场景** | 弱网测试：切 2G/3G/4G/5G/飞行模式；低电量测试：电量<20% 时的行为；中断测试：来电/消息通知/切后台再回来；不同机型/系统版本测试 |
| **Bug 追踪** | 跟进自己提的 bug：开发修了→回归验证→关 bug →没修好打回重开；每天和开发同步 bug 状态，P0 级 bug 当天必须修 |
| **回归测试** | 版本上线前全量回归冒烟（核心流程不能断）；自动化回归 + 手工补充 |
| **上线后** | 线上验证（灰度和全量后快速冒烟）；收集线上用户反馈的 bug，复现→定位→提交 |

### 🔧 常用工具
- Bug 管理：Jira / TAPD / Bugly
- 抓包：Charles / Wireshark / Proxyman
- 日志：Android logcat / iOS Console / 内部日志平台
- 截图/录屏：系统自带 + 标记工具

### 🏢 大厂差异
- **国内大厂**：功能测试团队独立，十几到几十人，外包比例大（腾讯/阿里的手工测试大量使用外包供应商）
- **谷歌/苹果**：手工测试人员极少，但极为重视自动化+开发者自测文化（Google 的 Testing on the Toilet 文化）；苹果有庞大但保密的内部 QA 团队
- **中小公司**：1-2 个测试，或开发自测为主

---

## 2. 自动化测试工程师

**一句话定位**：写脚本替代人工重复测试——每次发版自动跑，秒级发现回归 bug。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **UI 自动化** | 写 UI 自动化脚本（iOS: XCTest/XCUITest, Android: Espresso/UiAutomator）；覆盖核心流程：登录→浏览→下单→支付的回归用例；维护脚本——需求变更导致 UI 变了，脚本也要跟着改 |
| **接口自动化** | 写接口自动化测试（pytest + requests + Jenkins）；所有接口覆盖：正常返回 + 异常参数 + 边界值 + 未登录/无权限；每次代码提交自动跑，失败告警 |
| **每日 CI 自动化** | 每日凌晨自动拉最新代码 → 打包 → 安装 → 跑自动化 → 生成报告；失败用例自动截图 + 日志 → 发到测试群 |
| **维护优化** | 分析不稳定用例（flaky tests）的原因，降低误报率；和功能测试协作，把高频回归用例转为自动化（降低人工重复劳动） |

### 🔧 常用工具
- iOS：XCTest / XCUITest
- Android：Espresso / UiAutomator / Appium
- 接口：pytest / Postman / Newman / JUnit
- CI：Jenkins / GitLab CI / GitHub Actions

### 🏢 大厂差异
- **谷歌**：自动化测试文化最强，提交代码必须有自动化用例，覆盖率要求高
- **国内大厂**：自动化测试覆盖率在逐步提升但不如谷歌，核心链路 100% 自动化
- **中小公司**：通常无自动化测试

## 3. 性能测试工程师

**一句话定位**：保证 APP 不卡顿、不耗电、启动快——体验性能指标。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **启动性能** | 冷启动/热启动耗时测试（取 P50/P90/P95）；竞品启动速度对比 |
| **页面性能** | 页面加载耗时、首屏渲染时间、列表 FPS/卡顿率 |
| **内存** | 内存占用峰值测试（各页面遍历测内存）；内存泄漏测试（反复进出页面看内存是否只增不减）；后台内存占用（App 切后台系统杀进程阈值） |
| **功耗** | 各场景耗电测试（待机/正常使用/高强度使用）；定位/蓝牙/后台持续运行场景的耗电 |
| **流量** | 首次安装流量消耗；日常使用流量消耗 |
| **服务端性能** | 接口响应时间（P50/P99）、QPS 承载能力；全链路压测（模拟双11级别流量） |

### 🔧 常用工具
- iOS：Instruments（Time Profiler / Allocations / Leaks）/ Xcode Energy Log
- Android：Android Studio Profiler / Systrace / Perfetto
- 服务端：JMeter / Locust / 内部压测平台

### 🏢 大厂差异
- 大厂有独立性能测试团队，上线前必须过性能基准（劣化超标不准上线）
- 谷歌/苹果：性能测试集成到 CI，每次提交自动跑性能基准测试
- 中小公司：开发自己看 Instruments/Profiler

---

## 4. 兼容性测试工程师

**一句话定位**：保证 APP 在几百种机型上都能正常用。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **机型覆盖** | 维护测试机型库（低配/中配/高配/旗舰，覆盖主流品牌 TOP 机型）；核心功能在 TOP 50 机型上遍历 |
| **系统版本覆盖** | iOS 最近 3 个大版本（如 iOS 16/17/18）；Android 从 minSdk 到最新版本的关键 API Level 测试 |
| **特殊设备** | 折叠屏（展开/折叠态）、平板、刘海屏/挖孔屏/瀑布屏 |
| **厂商 ROM** | 小米 MIUI/华为 EMUI&HarmonyOS/OPPO ColorOS/vivo OriginOS/三星 OneUI |
| **兼容性自动化** | 使用云真机平台批量跑兼容性测试；截图对比（不同机型 UI 显示是否正常） |

### 🔧 常用工具
- 云真机：阿里云测 / 腾讯 WeTest / Testin / Firebase Test Lab
- 本地设备：自建机柜

### 🏢 大厂差异
- **国内大厂**：有自己的机柜（上百台真机），自动化兼容测试每晚跑
- **谷歌**：Firebase Test Lab 提供云端兼容测试，Android 兼容性是平台方视角
- **苹果**：碎片化远小于安卓，兼容性测试成本低
- **中小公司**：买几台测试机，主要靠云测平台

## 5. 安全测试工程师

**一句话定位**：模拟黑客攻击，找安全漏洞，防数据泄露。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **渗透测试** | 模拟攻击：抓包/篡改请求/重放攻击/SQL 注入/XSS/CSRF 等 OWASP Top 10；检查敏感信息是否明文传输/存储 |
| **APP 安全审计** | 逆向 APP 检查（是否可反编译出源码/密钥）；检查是否有 Debug 日志/测试接口泄露到生产环境；第三方 SDK 安全筛查 |
| **隐私合规** | 检查 APP 是否在未经授权时采集隐私数据（定位/通讯录/剪贴板）；隐私弹窗和权限说明是否符合法规 |
| **代码扫描** | SAST（静态代码安全扫描）集成到 CI，高危漏洞自动拦截 |
| **安全报告** | 输出安全测试报告，列出漏洞等级和修复建议 |

### 🔧 常用工具
- 抓包：Burp Suite / Charles / Wireshark
- 扫描：OWASP ZAP / Nessus / Fortify
- 逆向：IDA Pro / Frida / jadx / apktool

### 🏢 大厂差异
- **阿里/腾讯**：安全测试是独立信息安全部门，安全测试工具大量自研
- **谷歌**：安全团队文化深厚（Project Zero），自动化和人工结合
- **苹果**：对安全测试团队高度保密，但从 iOS 的安全性能看出投入巨大
- **中小公司**：无此岗位，请第三方安全公司做渗透测试

---

## 6. 合规测试工程师

**一句话定位**：保证 APP 不被应用商店下架，过审是第一要务。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **App Store 审核** | iOS 隐私清单（Privacy Manifests）完整性检查；审核被拒记录维护，总结拒审原因库；提交审核前把所有审核 Checklist 过一遍 |
| **Google Play 合规** | Target API Level 要求（每年升级）；数据安全表单填写审核；应用分级（IARC）、广告 ID 声明 |
| **隐私合规** | 隐私弹窗是否在合适时机弹出（使用前弹，不是打开就弹）；ATT（App Tracking Transparency）是否正确实现；用户是否有权删除账号/数据 |
| **广告合规** | 广告是否清晰标注（"广告"标签）；是否包含诱导点击/虚假内容 |
| **儿童保护** | 是否涉及儿童个人信息收集（COPPA）；是否有适龄指引 |
| **国内合规** | 用户协议和隐私政策是否可访问；注销账号功能是否可用；敏感权限申请是否符合「最小必要」原则 |

### 🔧 常用工具
- Apple App Store Connect / Google Play Console / 国内各应用商店后台

### 🏢 大厂差异
- **苹果/谷歌**：审核政策由平台方制定，内部合规团队天然强，和平台审核团队频繁沟通
- **国内大厂**：有专人对接应用商店审核，熟悉各家规则和「踩坑经验」
- **中小公司**：开发兼做，被拒了再去改
