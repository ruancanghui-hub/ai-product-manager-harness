# 六、项目管理 & 管理岗 — 日常工作总结

> 统筹进度、把控质量、对最终结果负责

---

## 1. 项目经理 PMO

**一句话定位**：APP 版本迭代的「列车调度员」，让所有人按时到站。

### 📋 日常核心工作

| 时间段 | 工作内容 |
|--------|----------|
| **每日** | 主持/参加站会（15 分钟），过每个人昨天做了啥、今天要做啥、有没有阻塞；跟进阻塞项——谁卡住了、需要谁配合、什么时候解决；更新 Jira/TAPD 看板，确保任务状态准确 |
| **每周** | 进度同步会——拉各端负责人过本周进度和风险；输出周报：本周期望 vs 实际进度、风险项、下周计划；资源协调——谁闲了、谁忙不过来了，调整人员分配 |
| **每迭代（2-4周）** | 迭代规划会——拉 PM+开发+测试对齐本期需求范围和工作量评估；排迭代日历：需求评审→技术评审→开发→提测→测试→发布；每日跟踪迭代燃尽图（Burndown Chart），发现偏差及时干预 |
| **里程碑节点** | 需求冻结日→代码冻结日→提测日→发布日；每个节点检查完成质量，不达标不开闸放行 |
| **风险管理** | 识别风险：第三方依赖延期、人员请假、技术难题卡住、需求变更；风险上升机制——超过 TL 能解决的升级到总监；记录和复盘风险处理经验 |
| **版本发布** | 发布前 checklist：代码合入完成/测试通过/性能达标/合规通过；发布当天值班：监控灰度数据、准备回滚方案；发布后复盘：这次延期了吗？为什么？下次怎么避免？ |

### 🔧 常用工具
- 项目管理：Jira / TAPD / Linear / Asana / Trello
- 文档/周报：飞书 / 钉钉 / Confluence / Notion / Google Docs
- 甘特图：OmniPlan / Microsoft Project / 在线工具

### 🏢 大厂差异
- **阿里/腾讯**：PMO 分组织级和项目级。组织级 PMO 管全部门项目组合（哪些项目做、哪些不做）；项目级 PMO 跟具体 APP 迭代
- **谷歌**：TPM（Technical Program Manager）偏技术项目管理，需要懂技术；项目管理文化比较轻量（OKR 驱动而非强计划驱动）
- **苹果**：EPM（Engineering Program Manager）是新产品的核心推动角色，权力大、责任重，直接向 VP 汇报；保密文化下 PMO 信息管控极为严格
- **中小公司**：PM 或 TL 兼做项目管理，没有专职 PMO

## 2. 技术负责人 TL / 架构负责人

**一句话定位**：管技术也管人——技术方案兜底 + 团队成长 + 线上故障最后防线。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **技术管理** | 评审重大技术方案（新系统/新模块的技术设计文档）；制定技术规范和编码标准（Code Style、命名规范、目录结构）；Code Review——关键代码必须 CR，保证代码质量不滑坡 |
| **团队管理** | 1-on-1 面谈——每周和团队成员一对一聊天（工作状态、成长困惑、职业发展）；绩效评估——半年/年度绩效打分、晋升推荐；招聘——面试候选人、定级、谈薪；新人 Onboarding——导师分配、培训计划 |
| **技术规划** | 技术债务管理——列出技术债清单，每迭代安排 20%-30% 时间还债；技术演进方向——微服务拆分、框架升级、存储升级等中长期技术规划 |
| **线上稳定性** | 值班制度——TL 通常是线上故障的最后兜底人；故障应急——收到 P0 告警→拉人→定位→止损→复盘；故障复盘——写 Postmortem（故障报告），定改进措施 |
| **跨团队协作** | 和 PM 对齐需求和排期（能不能做、做多久）；和其他 TL 对齐跨团队技术方案（接口定义、联调计划）；向上汇报——给总监汇报团队状态和技术演进方向 |

### 🏢 大厂差异
- **阿里**：TL 一般管 8-15 人，P8/P9 级别，技术和管理并重
- **腾讯**：技术 Leader（T9-T12），不同 BG 管理风格差异大（微信偏极客文化，游戏偏重交付文化）
- **谷歌**：Tech Lead 和 Engineering Manager 经常是两个人——TL 管技术，EM 管人，强调「技术和管理分离」
- **苹果**：Manager 角色重，管理链路清晰，保密文化下跨团队沟通限制多
- **中小公司**：TL = 最资深的那个开发，管技术也管人，甚至还得亲自写代码

## 3. 部门总监 / 业务负责人

**一句话定位**：APP 的最终负责人——定战略、管预算、担业绩。

### 📋 日常核心工作

| 工作项 | 具体内容 |
|--------|----------|
| **战略制定** | 定业务方向——这个 APP 要服务谁、解决什么问题、商业模式是什么；定年度/季度 OKR 或 KPI（营收/DAU/市场份额）；看市场趋势和竞品动态，判断要不要调整方向 |
| **资源管理** | 管人——团队招聘 HC（编制）、人员晋升和汰换、核心人才保留；管钱——预算申请和分配（人头成本、服务器成本、投放预算、外包费用）；管优先级——多个需求抢资源时，拍板做哪个不做哪个 |
| **向上管理** | 给 VP/CEO 汇报业务进展和业绩；争取资源——要更多 HC、更多预算、更多时间；展示成果——版本里程碑、业务指标提升、行业影响力 |
| **风险管理** | 重大风险决策——线上重大故障、PR 危机、监管风险；跨部门协调——和市场/销售/法务/财务等平行部门协作 |
| **组织建设** | 团队梯队培养——不能出现「单点依赖」（只有一个人会的东西）；团队文化建设——价值观落地、团队氛围、跨团队协作文化；流程优化——团队大了流程要跟上，小了要灵活 |

### 🏢 大厂差异
- **阿里**：M 序列（管理岗），P10+（资深总监/VP），考核重结果也重文化认同
- **腾讯**：GM（总经理）级别管一条业务线，汇报给 VP/EVP，重产品能力和战略眼光
- **谷歌**：Director/VP 管大团队（50-200+ 人），OKR 驱动而非 KPI，重视长期目标
- **苹果**：VP/SVP 权力集中，DRI（Directly Responsible Individual）文化——每个事必须有一个明确负责人
- **中小公司**：创始人/CEO 直接管，没有中间层
